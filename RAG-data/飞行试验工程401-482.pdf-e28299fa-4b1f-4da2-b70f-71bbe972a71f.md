飞行试验活动是整体组织的，准备阶段由制造商和试飞中心共同负责，另一个阶段由试飞中心单独负责，根据试飞中心的试飞结果形成使用该装备的官方命令，也就是定型。

这里的飞行试验包括飞行试验台试飞。例如，“幻影”Ⅲ战斗机的阿塔9K50（ATAR9K50）发动机的研制和定型中首先用Armagnac飞行试验台进行压气机的飞行试验和发动机/飞机相互干扰的飞行试验。

“幻影”2000战斗机的M53发动机的研制和定型中，在“快帆”（Caravelle）飞行试验台进行了发动机调节规律优化，加力燃烧室调整等内容试飞。

# 11.5.5 英国、德国、意大利的发动机试飞

“狂风”（Tornado）型战斗机是一种可变后掠翼的歼击机，装有两台RB199MK01三转子发动机。该发动机由英国的罗·罗公司、德国的汽车发动机和涡轮发动机公司（MTU）以及意大利菲亚特公司共同组成的涡轮联合公司研制的。它的研制计划和实施过程充分说明了飞行试验台和飞行试验在现代新型发动机研制中的关键作用。从首批试验用的RB199-01生产出来到完成发动机定型试飞（FQT）后定型，历时7年多。在这期间共进行了三次“火神”飞行试验台的试飞和四次原型机上的试飞。每完成一次“火神”飞行试验台的试飞就发现一批重大的问题，相应对发动机作一次修改。三次“火神”飞行试验台试飞相继形成RB199-02、RB199-03和最终RB199MK01三种修改型。每次原型机试飞都是以完成飞行试验台试飞为前提。

综上所述，世界航空大国的各种著名航空发动机的研制过程，基本上沿着一条共同的研制路线，有着许多共同点：

- 非常重视发动机的飞行试验，飞行试验周期与发动机的研制试验周期基本上同步；  
- 通常在第一台发动机运转试验后不久立即开始用飞行试验台和试验机进行军用规范要求的初步飞行评定试验，然后用原型机试飞进行发动机定型，有一些发动机甚至直接用飞行试验台进行定型；  
- 飞行试验台、试验机和原型机试飞是发动机研制和定型的必经阶段。

# 11.5.6 关于飞行试验台能力的讨论

飞行试验台几乎可在整个发动机生命周期内得到广泛的应用，而且安全性好，成本低，但它也有局限性。飞行试验台的局限性在于高度、速度范围小，测量参数少、精度低等。但

是由于技术进步有的情况已经不存在了，有的缺点是可以克服的。下面我们来讨论这一问题。

# （1）飞行试验台的测试技术

随着现代化测试技术的应用，测试参数的数量和精度已经不成为问题。现代的飞行实验室能够安装大存储的测试设备，它们实际上能够记录在试验台条件下录取的所有数据。

而且由于机内空间大，科研人员可以直接上机调试和数据处理，大大提高了试飞效率。对于小飞机改装的飞行台，也可以通过遥测传输，加强地面监控和实时数据处理。

# (2）飞行试验台的高度一速度模拟试验范围的扩大

通过对进入被试发动机的空气流的加温可以大大地扩大飞行试验台的试验包线。进气加温不仅提高了可模拟的飞行速度，同时还提高了可模拟的飞行高度。其原理是飞行试验台的来流本身具有很大的动压，由于进气加温已提高了的模拟飞行马赫数，要保持与实际飞行相同的进气总压就必须有更高的高度，也就是模拟的飞行高度也提高了。换句话说，虽然飞行试验台飞行速度比较低，但对进气受到加温的被试发动机来说，其进口总压和总温代表了较高飞行高度和较高飞行速度的模拟飞行状态。

以轰6飞机改装的飞行试验台为例，如果进气加温 $\Delta T = 50\%$ 时，最大模拟飞行马赫数可达 $M_a = 1.76$ ，最大模拟飞行高度可达 $H = 14650\mathrm{m}$ ；如果进气加温 $\Delta T = 80\%$ 时，模拟试验包线的面积相当于原轰6飞行包线的3倍，最大模拟飞行马赫数可达 $M_a = 1.96$ ，最大模拟飞行高度可达到 $H = 16500\mathrm{m}$ 。因此，经过进气加温的轰6飞行试验台的包线已覆盖了现代战斗机飞行包线的大部分最主要的区域。大大扩展飞行试验台的试验能力。

在飞行试验台上采用进气口加温是具有可行性的，可以采用的进气加温装置有：

- 氢燃烧直接加温；  
- 使用煤油的普通燃烧室的燃气进行加温；  
- 氢燃烧/热交换器；  
- 使用煤油的普通燃烧室/热交换器；  
- 混合加温装置。

飞行试验台的模拟试验包线的高速边界进行一定的扩展外，还可以向小速度边界移动。最简单的办法是在吊舱中加阻力网，如果降压系数低到 $\sigma = 0.8$ ，在相同的马赫数下小速度边界可以向高度增高方向提高 $1400\sim 1500\mathrm{m}$ ，如果 $\sigma = 0.7$ ，高度可以提高 $2200\sim 2300\mathrm{m}$ 。另外一种方案是，在进气道中安装风动涡轮，这种方法不仅可以有效地降低发动机进口总压还可以降低总温，小速度试验范围可以大大扩大。这种措施也可以用来进一步扩大整个模拟试验包线的高度范围。

为了保证发动机喷口能模拟较高的飞行高度，可以采用引射喷管。根据试验结果的计算表明，即使使用最简单的圆柱形引射喷管，控制引射系数 $K_{\mathrm{ej}} = 0$ （即没有二次流）在 $Ma = 0.8$ ， $H = 11\mathrm{km}$ 就可以使发动机喷口处的静压下降到等于 $H = 16\mathrm{km}$ 时大气压力。

迎角和侧滑角的模拟对温度-压力舱式地面高空模拟试验台是困难的，但对高机动性飞机的中、低空使用又是极为重要的。在飞行试验台上可以通过适当设计吊舱收放机构设置迎角和侧滑角，在一定程度上扩大模拟范围。

当然这里只是一些方案设想，具体发动机飞行台和具体被试对象不同，可以有不同的方案。总之随着科学技术进步，飞行试验台的试验验证能力可以进一步扩大，但必须看到，它

永远无法代替原型机试飞。

# 11.6 机载系统试飞验证平台

# 11.6.1 机载系统的发展和试飞验证

机载系统是一个非常广的概念，它包括了除飞机平台和发动机以外的所有飞机系统。因为现代飞行控制系统对飞机平台有极大的增强作用，有时把飞控系统列入飞机平台中去，但它又具有机载系统的属性。现代飞机中，由于航空电子和武器火控系统紧密融合，因此往往把两者统称为航空电子系统，它们是机载系统中分量最重、分布最广的系统。航空机载系统伴随着飞机的诞生而诞生，随着飞机的发展而发展。当航空科学技术出现大的飞跃发展时，机载设备也跟着出现了质的变化，由简单的机电和光电结构转向了综合化、智能化、模块化的复杂结构，而且这种综合化和复杂化的程度还在不断加强，在飞机总体中的位置和分量也在不断增强。甚至可以说，在航空技术和航空产品发展中，航电和其他机载技术和系统的发展更新率更快、门类更多，航电系统对于军机作战和民机运营效率乃至生存性的影响越来越显著。从航电系统占飞机总体出厂成本角度看，70年代F-15C占 $21\%$ ，80年代F-16C占 $30\%$ ，现代飞机已占到 $35\% \sim 40\%$ ，而F-22飞机机载设备总成本的比例已接近全机的 $50\%$ 。三种飞机航电系统研制费用比重如下图11-14所示。

总之，航电和武器火控系统的研制和试飞工作量越来越大，周期越来越长已成趋势。这种趋势往往与航空装备研制总体进度要求又相矛盾，如何摆脱这种困境，至少应从如下几个方面着手。

（1）加强航空电子设备或分系统的领先研制，促进它们的成熟性和通用性

每个机载产品都有其技术复杂性，都有其研制周期，因此在主机研制计划启动之前就应该安排它们的研制计划。只有这样才不至于把单个航空产品的不成熟性堆集到新飞机型号上。从科学发展的逻辑，它们的研发应领前主机一个周期，最好能成为各主机型号备选的货架产品。要做到这一点，除了航空产品研发本身工作外，事先通过空中飞行实验室的领先验证是非常必要的。

（2）充分的地面实验室综合试验

这种总体试验是全面的和一比一的软硬件综合的试验。对于具有几百万条软件语句的现代飞机来说，这种综合试验工作小时数应以万计。现代飞机不但要充分进行地面实验室综合试验，而且这种试验应尽早与空中实验室试飞并行。提早迭代，尽可能不要把空中试飞和地面实验室之间的优化迭代带到原型机的试飞阶段。

(3) 充分的空中实验室综合试飞

由于地面实验室的环境与空中实验室的环境毕竟不同，特别是大气环境、电磁环境的不同和目标模拟的空间限制，地面试验永远代替不了空中试验，尤其是对空、地、天、海全时空和宽频域网络作战体系的时代，空中试验更不可或缺。这种空中实验室的试飞小时数以千计。各国型号发展的例子表明，凡是经过较充分的他机试飞的型号，进展就较为顺利，否则就会事倍功半。这里空中实验室即本节所指机载系统试飞验证平台。

机载系统试飞验证平台是指专门用于探索和验证航空机载系统/产品科学领域新问题、新理论、新技术、新结构而专门研制或改装的飞行器。

航空事业的发展在很大程度上依赖于飞行实践，许多技术数据、物理现象（往往是关键性的）只有通过飞行试验才能获得、才能解释，计算再详细准确、地面试验再充分可以减少飞行试验的风险和压力，但永远代替不了空中的飞行试验验证。

机载系统试飞验证平台是用已有各类飞机改装而成的专用或通用试飞验证平台，但本节的重点是利用运输机型试验机构成的空中动态航空电子系统通用试验平台。该平台配备有用

于飞行试验的通信、控制、引导系统，安装有较全的试验基准系统和测试系统，并为被试产品提供必要的空中试验接口和环境条件。试验平台主要功能是用于航空电子系统新理论预先研究和验证、项目原理性演示验证、项目研制性试验、新产品鉴定和大系统调试/鉴定等。试飞验证平台为这些项目提供了真实空中环境下的测试、检验，从而进行必要的软硬件调整、系统原理确认、新技术验证、产品设计优化及设计缺陷改进，以及产品和系统鉴定等。试飞验证平台已成为机载系统/产品研发过程中不可缺少的试验手段，同时也为保障型号试飞安全、提高型号试飞效率、缩短型号研制周期、降低型号试飞风险和新型号研制风险都会起到不可估量的作用。

# 11.6.2 典型的机载系统试飞验证平台

试飞验证平台已经成为机载系统/产品研发过程中必不可少的重要条件。自从第二次世界大战以来，为适应雷达、武器火控系统试飞技术飞速发展的需要，在飞行试验中，国外已经开始使用大量的综合航空电子试验机，如美国怀特基地（俄亥俄州）、依格林基地（佛罗里达州）和爱德华空军基地中就拥有C-14，C-135，C-130，F-4，F-15，F-111，A-7，B-52等10多架航空电子试验机。西屋公司拥有A-3B，F-4B/C/D/E/J/K，BAC-11等10多种试验机。俄罗斯飞行研究院研制和应用了30多架航空电子试验机。俄罗斯明确规定不经过试验和验证的航空机载产品，不能用于新机研制。其他国家，如荷兰、英国、法国、德国等各自的试飞中心、宇航研究院等均拥有多种试验机。

机载系统的根本属性就是要在空中飞行环境下，正确完成既定的任务功能，并具备良好的技术性能。机载系统的研发成果只有通过试飞验证，才能够真正保证其有效性和实用性；通用试飞验证，能够及时发现和解决机载系统在飞行条件下可能出现的许多问题，提高机载系统的技术成熟度和可靠性，更好地满足飞机的配套要求。

# 11.6.2.1 波音757航电系统试飞验证平台（FTB）

F-22 航电系统试验的重中之重是软件的试验，其试验步骤是：系统软件各组成部分开发出来后，先在各软件开发实验室内测试；然后整个任务软件组合到航电综合实验室（AIL），初步进行硬件和软件的综合测试；AIL 综合测试完成后或初步完成后，整个软件包再装载到 F-22 航电飞行试验台波音 757（FTB）上集成，进行空中环境的综合试飞；最后航电软件在 F-22 试验飞机上完成最后的试验。

年即开始了波音757空中实验室初步航电试飞。1998年将F-22的航空机载系统安装到波音757飞行试验台，进行大量飞行试验和验证，直到2004年。也就是说，波音757（FTB）伴随了F-22整个研制过程。

在正常试飞情况下，F-22航空电子及试验设备由波音757的辅助动力装置供电。

F-22 航空电子系统在飞行试验台上的试飞计划分 4 个单元，采用积木式试验方法，综合试验逐步扩大，综合雷达、电子战和通信/导航/识别系统，依次进行数据融合。负责 F-22 航空电子的试飞队伍从 2000 年初开始试飞，波音 757 飞行试验台的试飞提前一年进行。二者是接力飞行试验，航空电子系统先经过波音 757 飞行试验台飞行试验考验、修改且成熟之后再装到 F-22 第 4 号试验机上试飞。由于已经在空中飞行试验台中进行了测试，航空电子在 F-22 飞机上试飞时间需要 $1700 \mathrm{~h}$ ，相比之下，如果不进行波音 757（FTB）试飞，这样复杂的航空电子在该机上的试飞时间需要 $4000 \sim 6000 \mathrm{~h}$ 。

虽然波音757飞行试验台进行了外部修改并加装设备，其飞行速度限制在马赫数0.64，但还能容易地达到 $4\sim 5\mathrm{h}$ 的试飞时间要求。飞行试验台也并非由F-22模拟座舱控制，仍由波音757驾驶舱中的客机驾驶员操纵飞行。飞行试验只是找出运动补偿方法，并非高速或拉过载，而是要看看飞机在转弯和面对多个目标时，航空电子系统究竟在干什么，传感器是否在跟踪。飞行试验台低速飞行试验是为F-22高速飞行试验做准备。飞行试验台可以使机上试验人员和软件工程师在空中实时搜集数据、发现问题、迅速分析甚至改写软件。即使遇到不能立即解决的问题，也能缩小范围，随后排除故障，从而节省了地面试验费用和时间。

F-22的综合航空电子系统是以“宝石柱”设计思想为基础，采用了“宝石柱”、IC-NIA、INews三项预先研究成果（见11.6.2.5节）的高度综合的航空电子系统。它的构型归纳为：通过数据总线进行信息传递；通过下显、平显和/或头盔瞄准具进行综合显示；采用模块化结构实现结构的简化和资源共享；通过传感器数据融合获取丰富、准确、高质量的目标信息；通过机内自检测（BIT）、系统重构和容错能力，提高系统可靠性和维修性。

为了使这样一套采用多项高新技术综合一体的航空电子系统的技术成熟和工作稳定，美国波音公司自1987年专门改装了一架波音757飞机，对其进行了系统的、有步骤的试飞测

试和验证，这套航电系统在波音757（FIB）上的验证试飞分为演示/验证试飞阶段和研制试飞阶段。演示/验证试飞阶段的试验目的是：验证系统预计性能是否可以达到，技术是否成熟、可用；而研制试飞阶段的目的是对航电系统软、硬件性能进行全面测试和验证。波音757（FTB）80年代末演示验证了多传感器数据融合；90年代验证了有源相控阵雷达、电子战和通信导航与识别（CNI）传感器；1999年3月开始F-22航电软件试飞，到2004年共完成飞行试验超过 $1200\mathrm{h}$ 。

然而，F/A-22的飞行试验计划在实施过程中仍然出现了严重的延迟，除了飞机制造拖延而造成的影响外，突出的问题是F/A-22航电软件的稳定性问题。这种不稳定表明软硬件出现异常，这种异常虽然不会影响飞机的飞行控制，但使得试飞效率大大降低。这些异常可以造成部分航空电子系统如雷达处于异常状态，甚至完全不能运行。在这种条件下，试飞员必须重新启动F/A-22航空电子系统。由于该机的航电软件异常复杂，虽然采用AIL、RAIL、FTB等进行了不少排故工作，但在F/A-22本机上试飞时还是遇到了许多问题。软件可靠性的检查、确认是非常困难的工作，从而引起F/A-22的试验进度一拖再拖。以至于2003年1月，五角大楼批准了8.76亿美元的F/A-22项目调整计划，将资金从采购账上转移到研发上，以支付因航空电子系统问题带来的超支，主要用于航电系统的鉴定试飞、改进和验证方面。

# 11.6.2.2 波音737航空电子飞行实验室（AFL）

由于F-22航空机载系统飞行试验台上的成功试验，波音公司又把此方法用于JSF飞机，而JFS的另一竞争对手——洛克希德·马丁公司也竞相效仿。由此而出现了波音公司用波音737改成的飞行实验室（Avionics Flying Laboratory）和洛克希德·马丁公司用BAC-111改成的综合航空电子试验台（Cooperative Avionics Test Bed）两架JFS航空电子飞行试验台。此外，欧洲战斗机“台风”计划也得益于飞行试验台，用BAC-111来试飞CR90雷达的远距探测和跟踪性能，削减了开发费用。为了F-35航电系统的综合飞行测试，洛克希德·马丁公司从德国汉莎航空公司购买了1架波音737-300飞机，将其改造成“航电系统的综合试验与测试平台”。

F-35的工程和制造发展（EMD）定于2001年初开始，2012年结束，2006年首飞。1999年起BAC-111航空电子合作试验平台（CATB）和波音737飞行实验室（AFL）进行了航空电子系统和关键传感器的飞行试验。

波音公司使用的（AFL）是1架高度改装的波音737-200，在航空系统装入JFS前进行试验和故障诊断。这架飞机最大装载15t，最大起飞重量104t。该AFL上承载了一些军用航空的最先进技术，帮助JSF减小试验风险，并降低JSF项目的费用。AFL验证了一些JFS系统，包括核心处理器、多功能雷达和其他传感器。AFL在预先综合和预先试验JSF航电系统方面为波音提供了高度的灵活性。AFL还可以承载大量的航电工程师对试验数据进行实时监控和排除系统异常。而常规的战斗机平台要求工程师只能在地面上，依靠遥测和再次处理试验数据来分析试验结果。

# 11.6.2.3 多电飞机验证计划

美国多电飞机计划是一项旨在取消液压作动系统的重要计划。由于它直接关系到飞行安全，因此对此项目的应用持非常慎重的态度。从80年代开始有许多试飞研究项目支持了这样计划，最后初步用在了“JAST（联合先进攻击技术）”计划之中。

1994年1月，美国国防部建立了一个联合军种组织，即联合先进攻击技术（JAST）计划组织，确定研制用于未来战术飞机的武器系统和任务技术。该计划的目标是给下一代战术飞机的研制进行技术储备。为了实现这个目标，联合先进攻击技术计划组织与宇航工业部门和美国空军以及美国海军的科学技术组织一起确定了联合先进攻击技术计划的研究，选择了四种技术作综合评估。这四项技术包括：多电飞机技术、机电系统综合技术、先进飞机管理系统和先进固定面积喷管技术。通过综合方法，将这四种技术的最佳特性结合，在JAST项目的时间框架内可以获得具有革命性的的技术特性以及可观的收益。

联合先进攻击技术计划办公室建立了一个美国空军和海军的联合研究计划，即联合先进攻击技术综合机电系统技术（J/IST）验证项目计划。该项目就是要推动完成使综合机电系统技术成熟的一些必要的研究活动。联合先进攻击技术综合机电系统技术验证计划就是要让工业和政府决策者确信综合机电系统技术会使寿命周期成本降低，提高战斗能力。

J/IST 验证系统计划共包括对 JSF 计划中的 56 项技术进行试验验证，并最终集中进行四种飞机级的飞行验证来检验 JSF 所需的综合系统技术。这四种飞机级的一体化技术项目包括热能量管理组件 T/EMM 系统和供电系统。T/EMM 系统的飞行试验将单独验证 8~10 种 T/EMM 系统的运行模式，然后与普惠提供的、与未来 JSF 可能选用的发动机进行一体化工作，并再进行试验验证，以评估该 T/EMM 组件在整个 JSF 飞行包线内的运行模式和瞬态响应。

多电飞机技术最关键的是功率电传作动器，用以代替传统的液压作用器。事实上，功率电传作动器项目从1981年就开始启动。主要的飞行试验计划有C-141飞机功率电传作动器的飞行试验计划、C-130电作动系统计划以及联合攻击战斗机/综合公共设备系统技术验证计划等。其中最早的一项飞行试验是在1986年首次试飞的。90年代初，以美国空军为主，多电飞机计划（MEA）开始实施，该计划涉及发电、容错配电、电力管理、电防冰、电刹车、电力作动和发动机等诸多领域。

根据EPAD（电力传动设计）计划，美国NASA、美国空军、美国海军于90年代中期选用F/A-18为试验机，实施EPAD验证计划。EPAD的目的是验证功率电传作动系统。如果试飞验证成功，功率电传作动系统将取代传统的液压作动装置，并最终淘汰飞机上的集中式液压系统。

为此目的在NASA的F/A-18SRA系统研究机上，把左副翼上的标准液压作动器用EMA代替，并完成SRA飞行包线内的全部飞行，记录了各种数据，对左右机翼上的EMA

和液压作动器的全部性能作了比较。飞行试验结果表明EMA的性能非常接近所替代的液压作动器。

20世纪90年代，在AFTI/F-16上开展了为联合攻击战斗机（JSF）开发的J/IST验证试飞工作。AFTI/F-16验证了热能量管理组件（T/EMM）和包括270V直流电源以及分布式液压系统等新技术综合。AFTI/F-16的J/ISF计划证明了与使用常规机载机电系统的F-16相比，使用综合机载机电系统的F-16所获得的效益是非常显著的。

以后勤保障为例，现在24架F-16的保障设备需要16架C-141大型运输机来运输，而通过使用机电系统综合技术，原先用于F-16的大部分地面保障设备都可以省去。

随着机电系统综合技术的反复验证，证明与使用常规机载机电系统的F-16相比，采用综合机电系统技术的F-16的子系统零件数量可望减少 $40\%$ ，因此采购成本降低了 $3\% \sim 5\%$ ；由于减少了维护需求和零件数量，使寿命-周期成本节约了 $3\% \sim 4\%$ ；飞机最大起飞重量减少了 $5.5\%$ 以上；飞机体积缩小了 $9.1\%$ ；航程增加了 $20\%$ 。图11-22为AFTI/F-16验证平台，它为JSF/F-35的研制成功打下了坚实的技术基础。

美国在第四代战机的研发过程中，在80年代初期就提出了超级组合动力装置（SIPU）研究发展计划。在1986年中期，以AFTI/F-16作为技术验证飞行平台，进行了大量试飞验证（包含驱动270VDC电源系统），所得结果直接用于YF-22飞机上所装组合动力装置产品的研制上。该产品在YF-22飞机的试飞过程中取得良好的效果。

# 11.6.2.4 防护救生装备系统试飞验证平台

防护救生系统的试飞验证主要是针对飞机机动飞行状态下火箭弹射座椅的舒适性、约束效果、操纵可达性以及飞机制供氧系统的性能、个体防护装备的性能等进行飞行试验验证。应急救生情况下的试飞验证，主要是对火箭弹弹射座椅的性能、个体防护装备（主要是头盔、面罩等）的防护效果和应急氧气系统的功能进行验证，一般在专用空中弹射试验机上进行假人实际弹射。

通常除了飞机运动参数的记录外，弹射试验的假人身上要安装专门的测试系统，测试记录人体的各种相关数据，如过载、应力等，同时还要对座椅弹射轨迹，人椅分离姿态和容差等进行外部参数测量，以便对人机相容性、人椅相容性、椅机相容性等进行分析。

马丁·贝克公司在20世纪40年代末研制弹射座椅之初，就建造大量的试验设备，其中，位于德汉姆总部西北部的查尔格罗夫试验场有一个专用机场，经过改装的“流星”空中弹射试验用飞机（图11-24），可以进行 $834\mathrm{km / h}$ 的空中弹射试验。目前国际上大量装备MK10、MK12、MK14、MK16系列弹射座椅均在该试飞验证平台上进行了大量的弹射试验，包括相当数量的真人弹射试验。自1946年7月首次真人弹射试验以来，仅班尼林奇就在“流星”空中弹射试验机上完成了不少于16次的真人空中弹射试验，为马丁·贝克公司弹射座椅的高可靠性奠定了坚实的基础。

# 11.6.2.5 其他试飞验证计划

除上述的试飞验证项目外，世界上还有许多其他机载系统试验验证机项目，这里仅对美国其他验证项目做些简介。

# （1）“宝石柱”计划试飞验证平台

“宝石柱”计划是80年代中期由美国空军莱特实验室提出的先进的综合航空电子系统计划。这个计划与美国空军的先进战斗机（ATF）直接相关。它的主要目标是：

- 将航空电子系统按功能划分为几个标准模块，以增加容错冗余，系统重构能力和适当的降级工作能力；

- 增加航空电子系统的综合化水平，从而提高信息的关联性、准确性、容错能力和传感器资源共享，同时减少对冗长传感器的需求；

- 提高功能可用性和任务有效性；

- 将超高速集成电路（VHSIC）元件技术用于高速处理和总线接口，将高可靠性的元件级自测试能力用于方便和准确的在线错误隔离；

- 采用高速（50Mb/s）宽频带的光纤多路数据分配总线；

- 使用海量存储器进行大量数据存储及数据备份；

- 降低航空电子系统的费用。

“宝石柱”计划系统包括机电子导航、飞行控制、目标捕获和传感器数据合成（数据交联和融合）、武器管理和综合化座舱显示系统，整个系统的功能和重构边界划分为三个区，即：任务管理区、传感器管理区、飞行管理区。

# (2）综合通信、导航和识别航电（ICNIA）计划试飞验证平台

ICNIA计划的目的是由美国三军联合研制一种高度综合化和数字化的航空电子系统。它的概念是用标准化方法把外场可更换模块（LRM）划分为数量少、资源共享和可重构的通用模块类型。这些模块分为射频组，信号处理组和数据处理组，并可在软件的控制下实时组合完成通信、导航、识别和其他各种功能，关键模块有余度备份。采用这种通用模块可使ICNIA处理机的体积减小5/6，重量减少3/4。

# （3）综合电子战系统（INEWS）计划试飞验证平台

INews计划的目的是由美国空军和海军为下一代军用飞机联合研制综合核心航空电子和任务电子战系统INews可防御雷达发射的连续波、脉冲、微波和毫米波以及激光和电光/红外信号对航空电子系统的干扰。它由导弹告警系统、激光告警系统、雷达告警系统和先进的投掷装置等组成，INews的主要任务是接收、鉴定和处理发射源的告警信号，并根据干扰信息库中欺骗性和投掷式干扰方法启动响应信号，最后产生回避机动或与其他飞机协同干扰的指令数据。

INews 是能自检测、容错、故障弱化和抗战斗损害的航空电子系统，它不是一个装在飞机中的黑盒子，而是一套相互联系具有特殊功能和物理特征及接口的模块和系统。由于采用了光纤作为传输介质而获得了高抗电磁干扰能力，减轻了重量，提高了速度和容量。

# （4）高压液压系统试飞验证平台

$35\mathrm{MPa}$ 高压液压系统已经用于空客A380和波音787飞机， $21\sim 35\mathrm{MPa}$ 的变压力液压系统（双压力）已在F/A-18E/F中采用。为验证压力提高后液压系统的变化，1978年4月，利用T-2C教练机进行验证试飞。T-2C原来的液压系统经过了改装， $56\mathrm{MPa}$ 的液压系统

只用于横向操纵。其中右发动机上的 $21\mathrm{MPa}$ 液压泵仅提供升降舵、起落架、拦阻钩、减速板等操纵所需的液压功率。原来左发动机上的 $21\mathrm{MPa}$ 液压泵由 $56\mathrm{MPa}$ 液压泵代替， $21\mathrm{MPa}$ 液压副翼作动器也改为 $56\mathrm{MPa}$ 的作动器。原有的油箱及供油导管不变，为 $21\mathrm{MPa}$ 和 $56\mathrm{MPa}$ 两种系统所共用。

飞行试验总共11.5小时， $56\mathrm{MPa}$ 系统未发生任何故障，驾驶员反映良好。 $56\mathrm{MPa}$ 系统的工作特性与T-2C飞机上原来的 $21\mathrm{MPa}$ 横向操纵系统很相似。油液温度正常，压力脉动很小，产生的油液污粒也很少。后来又进行了 $10\mathrm{h}$ 的 $56\mathrm{MPa}$ 液压作动器操纵的方向舵飞行试验，试飞结果令人满意。

地面方向控制的研究在国外起步较早，是一项成熟的技术。英国的“EAP”和欧美的研究机构在20世纪80年代就已经开始进行前轮转弯的综合控制系统试飞验证。美国空军和波音军用机公司于1985年进行了综合控制模拟试验，仅使用飞机上产生的信号和设计要求，所需信号和可提供的控制都可以利用现有机载设备。着陆进场时，飞行员根据输入所期望的跑道方向以及预定的摩擦条件，使飞机在自动控制状态下进入着陆滑跑。当飞行员要转离跑道时，可以断开自动系统，手动控制飞机。

# (5）电子类产品试飞验证平台

1987年，B-2战略轰炸机的APQ-181相控阵雷达安装在KC-135试验机上进行雷达的空中试验，经过4年左右的空中验证试飞，使该雷达在功能、性能和关键技术上进行了充分的验证。1991年装有APQ-191相控阵雷达的B-2（AV#2）飞机首次飞行，到1996年才完成全部航空电子系统的飞行试验。

诺斯罗普·格鲁门公司有一架BAC-111客机作为雷达试验机，用来对处于原型机阶段的雷达进行原理验证飞行。F-22的有源相控阵雷达APG-77就是在这架飞机上进行验证试飞的。而F-35的航空电子系统综合试验和APG-81雷达的试验也是在BAC-111雷达试验机上进行。通过大量的飞行试验早期发现、排除F-35航空电子系统和雷达设计过程中的硬件和软件问题，极大地降低了飞机研制风险。

# (6) 综合环境控制系统试飞验证平台

美国海军的EEECs（能效环控系统）计划选定P-3侦察机作为综合环境控制系统的验证平台。EEECs于1982年9月开始实施，到1985年9月结束。全计划共分五个阶段，分别验证定速运转的蒸发循环压缩机和270V直流电源驱动的变速蒸发循环系统。飞行试验中收集的数据表明系统的性能和设计一样。尽管在前两次飞行的初期都发生过多次停车，但是，重启后都能正常工作，整个飞行试验的过程中，座舱温度都是舒适的（ $23\sim 27^{\circ}\mathrm{C}$ ）。

该计划验证了蒸发循环系统能够在飞机上成功地运行，并能够提高航空电子的可靠性，节省不少的功率。

# 11.6.2.6 中国试飞验证平台研发情况

国内在试飞验证机的设计、制造和使用等方面已经积累了不少的经验。在航空发展和飞行试验过程中已经使用了近20种各类飞行试验验证机，其中典型的有：IFSTA空中飞行模拟试验机、“飞豹”应急动力系统EPU专用吊舱试验机、歼教6弹射救生试验机、运7电子试验机、运8雷达电子试验机等多种飞行试验平台。具备了建立机载系统/产品研发试飞验证平台的技术和物质基础。为军工产品科研成果的工程转化和促进技术成熟方面发挥了不可忽视的作用。其中运8型雷达电子试验机作为雷达飞行试验平台，有以下技术特点：

- 良好的雷达试验环境：试验机内部空间大，不仅可满足不同雷达的改装要求，同时也可加装大量的试验支持设施，以满足不同雷达的使用要求，具有较好的通用性。  
- 完善的测试和调试条件：机载产品通常需要经过调整试飞，待性能稳定后方可开始定型或鉴定试飞。由于现代雷达技术的复杂性，使得调整试飞的难度和周期大大增加。而试验机由于得天独厚的条件，可以加装大量的专用测试设备，同时工程师可直接上机参与调试工作，这些都为雷达调整提供了优越条件。  
- 更长的留空时间：飞机采用的成熟技术，可以使其保持较高的出勤率，同时试验机留空时间长，在一个飞行场次中可完成更多的试飞科目，极大地缩短了雷达试飞周期。  
- 有良好的人机界面：不但保证试验人员对被试系统和测试系统的操作，而且保证试验人员对配试目标机的监控和调度。

# 11.6.3 机载系统试飞验证平台的作用

纵观国外机载技术试飞验证平台情况，结合我国机载技术的发展，机载系统试飞验证平台的作用如下：

# （1）机载系统新技术发展必须经过试飞验证

机载系统的测试首先是在地面上进行。由于机载系统的发展，机载设备越来越多，系统越来越复杂，地面试验可完成的检验相当有限。例如，为了验证综合航电火控系统各设备、各传感器是否能协调工作，共同完成复杂的功能，有必要在原型机试飞之前，将地面综合好的航电火控系统放在空中进行测试，为系统提供一个动态的，真实的飞行环境。在试飞验证平台进行系统的试飞，可以大大减少该航空电子系统在型号飞机上的试验时间。

按照科研流程，首先对关键技术进行建模仿真，并在射频仿真实验室进行相关的试验研究。但鉴于实验室条件的限制，以及目标特性、杂波特性、电磁环境等仿真模型局限性，实验室研究与空中试飞试验相比还有较大的差距，必须通过空中试飞这样的真实的试验环境进一步验证和研究，从中找出产品设计过程中存在的问题，并进行改进和完善，以提高关键技术和产品的性能、成熟度和实用性，满足战技指标要求。

在真实的环境中，常伴有各种干扰，目标也会出现抖动、闪烁、机动、间距密集、交叉飞行各种情况，使传感器出现虚警、漏警，导致探测数据不完整、不精确、模糊、甚至有假跟踪航迹，这种干扰和动态特性对传感器造成的影响在仿真或地面试验中是不能获得的。为全面验证、评估机载各传感器以及数据融合/传感器管理系统的功能性能，有必要进行试飞验证，使系统设计人员和软件工程师在空中实时搜集数据，发现问题，迅速分析，修改软件。

机载系统对大气数据系统的要求也在不断提高。如此高的精度要求，仅靠地面静态校准是无法保证的，总压、静压、迎角、侧滑角、总温实际使用是在空中和动态条件下，只有靠精确的试飞校准，得出全马赫数、全高度范围的误差曲线（校准曲线），由机载计算机进行修正，才能达到指标要求。

对机载系统/产品的研发工作，如果在主机机型确立以后再确定机载系统/产品的研制方案，机载系统/产品的研制深度和研制进度受到主机研制进度的制约。机载系统/产品一方面不能在机载设备研制的前期阶段进行试飞试验，技术问题无法在早期暴露发现，随主机试飞后，一旦出现问题，重则影响飞机安全，轻则也会对总体试飞工作造成影响，加大主机的工

作难度；另一方面，随总体试飞时，对具体机载设备针对性不强，试飞试验考核内容不够全面、深入，量化指标不具体，不利于把握机载设备真实的能力、水平，不利于发现产品问题，不利于其技术发展和提高。建设机载系统/产品试飞验证平台，可让机载设备厂（所）全力地进行产品预研工作，以使机载产品研发工作提前于主机的研制，整体上缩短主机的研制过程。

# (2）加快机载产品研发，促进成果转化

当今航空机载系统技术的发展和产品的更新换代非常快，而试飞验证又是迭代周期最长的环节，例如，一型新雷达的验证试飞可能需要2~4年周期，如果没有通用的试验平台，或临时寻找和改装验证平台，必将严重阻碍了航空机载设备技术的发展，最终将成为航空发展的瓶颈。通过飞行试验验证各项科研成果、新技术、新系统及其分系统的成熟程度和应用于武器装备的可能性以及节省费用的潜力。在空中真实飞行的条件下，一旦验证的新系统、新能力获得成功，系统就具备一定的工程能力，可尽快转入型号工程研制。

民用飞机的机载产品有适航性验证的要求。民机产业想要进入国际大市场，就需要研制过程中进行大量的适航认证试飞试验（TSO 验证）。机载产品研发试飞验证平台的建立，可以为机载设备厂（所）提供试飞试验的场所，可以在适航认证工作前做大量的准备和积累，甚至可使这些产品提前得到的适航认可，及早投入市场，供各主机选用。

# （3）新型号机载系统研制必需的环节

新一代型号飞机的机载系统技术跨度大、新成品比例大、综合化程度高、要求研制周期短，前期如果不投入较充分的试验机进行验证试飞，型号研制会遇到的麻烦是不言而喻的。而机载系统的工作性能与其他机上系统和飞行环境密切相关，因此必须在真实大气和硬件环境下进行广泛的试验，以验证和测试机载系统设计要求。

现代机载系统的结构体系发生了很大变化，如新的供电体制和制冷技术；航电系统则由主模式管理发展为分区管理等，功能、性能有了大幅度增加和提高，新技术新原理有了重大突破，例如：

- 齐平式大气测量系统；  
- 各种新型功能总线；  
- 远距、多信道、大容量、通信能力；  
- 远程高精度综合导航能力；  
- 智能化的座舱人机交互界面；  
- 综合电子战能力；  
- 远距单目标和多目标攻击；  
- 近距全向、大离轴、过失速机动攻击；  
- 空面精确打击；  
- 多机编队协同作战；  
- 联合体系作战；  
- 多传感器信息获取、融合、管理和目标识别能力等。

机载系统结构的变化和能力的提高，带来了研制风险的加大和软件开发的复杂性。单个设备的运行良好和基于各种假设的地面仿真试验，无法从根本上解决机载系统在真实空中飞行条件下诸如大气、地杂波、电磁、振动和人机交联等综合因素作用下才能暴露和必须解决

的问题。化解风险的措施之一就是贴近真实环境的验证试飞。

大型客机对航空技术是一个新的挑战，其中航空电子技术是非常重要的组成部分。与战斗机相比，大型客机航电系统有其特殊性，尤其是通信导航、交通管理、人机界面、防冰除冰、雷电防护，以及安全可靠、环境舒适性、技术成熟性等方面有更高的要求，同样需要领前试飞验证。

综合起来，机载系统试飞验证平台的作用可归纳如下：

- 用来研究和解决通过其他手段（如地面模拟试验、理论分析、数值计算等）所无法研究和解决的问题，通过机载系统试飞验证平台的飞行暴露理论分析和设计中没有发现的问题。  
- 使试飞员真实感受新技术对飞机性能及任务能力的改进程度影响，参与新技术研发，并进一步挖掘新技术的潜力。  
- 试验结果数据可用于支持实验室建模和设施验证，进一步优化和完善虚拟试验、建模和仿真技术等手段，提高地面仿真结果的可信度，极大地节省人力、物力的费用。促进设计技术、试验和测试技术的发展，包括设计工具、试验设施、建模方法、试验方法和检测方法等。  
- 机载系统试飞验证平台可以验证全系统的功能、性能、完整性以及系统综合集成技术和系统样机的设计水平。加快技术及早转化为工程能力，为新出现的重大关键技术攻关提供技术验证支持，带动和促进相关技术的发展。  
- 机载系统试飞验证平台的建立，可为大型民机机载设备和分系统进行领前的、独立的适航取证试飞。  
- 机载系统试飞验证平台将大大减少飞机机载系统/产品的研制成本，缩短机载系统/产品的研制周期，降低了机载系统/产品在新机型中应用的风险。  
- 对新系统和新产品在验证平台上的试飞，可以摸索新的试飞方法、测试和数据处理方法，培训试飞员和技术队伍，为型号飞机设计定型和适航取证试飞积累经验。

# 11.6.5 多机协同攻击/防御信息网和数据链系统演示验证

# （1）技术概念

随着信息时代的到来，空、天、地、海、磁五位一体的体系对抗已成为未来发展的趋势，战争模式也从平台为中心向以网络为中心转变。航空电子系统作为各种作战飞机的信息平台与火力指挥控制中枢，是网络中心战的信息网、探测网、作战网和指挥网的有机组成部分，是指挥、通信、控制、计算机、杀伤、情报、侦察与监视联合作战网络的关键节点。发展和装备飞机与飞机、飞机与地面、飞机与武器及飞机与卫星互联、互通和互操作的信息链路，是快速提高航空装备整体作战效能的倍增器。

从试飞验证的角度出发，其技术重点和难点主要体现为：

- 多机协同、编队和综合指挥、引导、作战规划和效能评估技术；  
- 中远距精确制导武器的数据链制导功能和性能试飞评估技术；  
- 飞行间数据网截获概率试飞评估；  
- 复杂信息环境下的数据传输能力、抗干扰能力、保密性及其可靠性等指标的试飞评估；  
- 大型民用机场和未来民用航空交通管理中的指挥、引导、监视、管理和航路分配等。

# (2）国外发展情况

美国F-22配置了内部数据链、联合战术信息分配系统（JTIDS）数据链和敌我识别数

据链系统，机间数据链（IFDL）使所有的F-22飞机在不使用语音通信的情况下，能分享飞行中的目标和系统数据。F-35在F-22的基础上，进一步对数据链通信天线、处理单元和软件进行了综合优化，实现了智能管理进一步对数据链通信天线、处理单元和软件进行了综合优化，实现了多波段、多模式的无线电信息通信，形成了战术数据链、JTIDS、战术因特网（TI），将双机、机群，甚至地面指挥中心、前沿观测站、地面部队、舰队、卫星等连成巨大的信息网络，实现了实时、自动、保密、抗干扰地交换信息、发布命令、发送控制信号，以致产生了全新的作战思想和战术。美国这种优势是以大量的投资和长期的研制和试验为前提的，其目的是为了控制现代战争的速度及复杂性，使军事指挥人员获得空前的态势感知。

美国协同交战能力（CEC）的第一次技术演示在1989年进行，开发性试验在1993—1994年进行，舰载型设备的初始作战能力于1996年在“休城”和“维克斯堡”巡洋舰上实现，这两艘巡洋舰试验了涉及不可靠跟踪数据的问题。1997年7月，在两栖攻击舰“黄峰”号上进行舰载设备的初始作战试验和评估期间，当舰上的操作手同时操作先进作战指挥系统和战术数据链时，遇到了很大的互操作性困难。在控制台上，操作手对不一致的数据、警报和识别冲突无所适从。1998年初，在“休城”和“维克斯堡”巡洋舰上进行的CEC与宙斯盾系统的试验出现了类似的互操作性和作战系统问题。CEC的机载型设备于1995年进行了空中开发试验，并于1998年在E-2C“鹰眼”预警机上进行了飞行试验。

在后来的多次试验中，综合问题逐一得到解决。而且在2000年9月和12月又进行了一系列海上试验，演示了装备和CEC的驱逐舰、巡洋舰、航母、两栖战舰以及飞机在内的实际战斗群可操作性。2001年5月，CEC进行了作战鉴定，8月发布了作战鉴定的结果，目前，已有几十个CEC系统在美海军中使用。为了进行试验，这些系统部署在各种不同的平台上，其中包括“提康德罗加级”、“阿利·伯克级”、“尼米兹级”、“黄蜂级”舰艇和“鹰眼”飞机，以及陆基设备（“爱国者”）等，美海军预计将大约200个平台装备CEC。可见美国在该项技术的研究和发展，经历了十多年的试验、试飞和改进。

# (3）试飞验证方案

多机协同攻击/防御信息网和数据链系统的研制和试验试飞验证是航空电子系统发展的必由之路。进行这类试验试飞的方法是利用现有飞机和空地通信设施改造和改装新一代数据链传输设备，构造适当的信息网、探测网、作战网和指挥网系统技术演示和验证的飞行和信息平台。

作为单机或网络节点的飞机可作为航电系统综合化技术验证的飞行平台，验证内容包括核心处理技术验证、综合驾驶全景显示技术验证、头盔视听说综合技术验证、综合光电通信导航、制导技术验证等，尽可能实现一机多用。多机协同攻击和防御信息网和数据链演示验证技术研究以当前已有的多种飞机为基础，适当改造和增加一些地面指挥、通信、干扰设施，联合各部门在系统集成、研制、试飞和保障方面的优势，确保项目的可执行性。

必要时可通过空中飞机平台和地面站中转传递，或制造仿真信息，包括威胁模拟和友机支援等信息参与体系试飞验证，形成空空、空地、空海数据链系统和产品验证试飞的基本环境条件，实现通信端机数据传输的速率、带宽、距离和可靠性等基本技术要素的试飞评估和鉴定。

采用试飞测试多信息源融合技术，解决数据一致性和精确性以及时统的问题，达到满足

新一代传感器产品和战术信息系统精度试飞和验证的要求；采用iNET增强遥测综合网技术，提高遥测传输率，提高中心站信息综合能力，实现全场域、全区域乃至更大范围的信息综合和资源分配。

通过基于三维的假定路线与合成路线的多目标、信息和轨迹管理和分配技术，大幅度改善和提高试飞和演示验证的技术和水平。并在此基础上进一步实现复杂信息环境下强对抗模式的多机协同攻击和防御信息获取、筛选技术和对抗策略的试飞评估；综合引导、指挥技术和作战思想的研究，实现包含武器在内的系统网络节点互操作、互探测系统的演示和验证。

选择一架大中型运输机作为中心指挥机和测试机，选择具有先进航空电子系统的若干飞机，在飞机上改装和落实数据链通信端机设备，在地/海面设置台站，即可实现同型战斗机之间、异形飞机之间、指挥机与战斗机群之间以及空中与地面站之间的数据链通信与显控。利用运输机的装载空间优势，可安装数据链系统的测试、仿真显示和控制设备，允许试飞、研制和指挥引导的技术人员直接参与和实时监视数据信息的形成、传输和接收应用的过程。

所谓多机协同包括有人机、有人机和无人机，以及无人机与无人机之间的协同。

应该指出，网络体系作战试飞是一个庞大的全局系统工程，这不是试飞工程师可以掌控的试飞验证项目。但是，作为体系中的要素、节点和单元，飞机本身参网组网功能和性能必须得到相应的验证。这就要根据现实条件，包括机种和数量，试飞场地和协同试验的设施，抓住关键技术，制定切实可行的多机协同攻击/防御信息网和数据链系统演示验证试飞方案，这是试飞工程师的职责。

# 11.7 本章小结

（1）试验研究机分为探索性研究机、技术验证机和空中实验室。作为航空科技发展的重要手段，试验研究机引领着航空科技发展，孕育了一代代新机的诞生，促进了航空新技术和新产品的成熟。自第二次世界大战以来，世界上各航空大国无不拥有大量的试验研究机，它们的研究成果为航空科技和航空工业的发展做出了巨大贡献。  
(2) X 系列研究机的发展取向往往代表着世界航空航天最前沿的概念和技术。其中最著名的有 X-1 突破声障, X-15 实现有人驾驶的临近空间飞行, X-29 重新启动前掠翼研究, X-31 过失速机动战术应用, X-36 无尾飞行器的探索, X-43 高超声速计划, X-45 和 X-47 无人作战飞机的起步, 以及 X-51 高超声速飞行器中引气冲压发动机持续工作的成功等, 它们大部分都具有里程碑的意义。  
(3) 空气动力是航空航天技术发展的先行学科, 风洞试验和CFD必须与气动力飞行验证相结合。气动力试验机应着重于雷诺数影响、尺寸效应、飞机/发动机一体化、超临界机翼、高升力多段机翼、大迎角、声爆和流动控制等方向的试飞研究, 因为这些方面风洞试验

难以给出精确的结果。

(4) 飞控技术的验证—试飞是试验研究机最活跃的领域之一, 它包括空中飞行模拟和专用的飞控试验机。从试飞观点, 飞控技术研究往往与飞行品质研究综合开展。世界上所有的电传操纵飞机没有不经过试验机领前试飞验证的。从美国飞控系统技术发展历史看, 几乎每一项新技术都要经过反复的试验机验证, 如 FBW、主动控制、大迎角、过失速机动、电操纵作动器、侧杆、光传、智能控制和综合控制, 等等。美国空军的每一种新机都要经过空中飞行模拟。美国航空科技在世界上领先, 但先进飞控技术在型号上的应用始终是慎之又慎。  
（5）航空发动机是极其复杂的高技术产品，是集流体力学、热力学、结构力学和电子控制于一体的热能机械。为此新发动机研制过程中，除地面试验外，飞行试验是必需的环节，包括通用发动机飞行台上的试验和原型机上的试飞。因此所有航空发动机大国毫无例外地都拥有相当数量的发动机飞行台。通常发动机飞行台应该按发动机种类和大小不同进行分类，因此，一个航空大国仅有一二架发动机飞行试验台是不够的。在发动机飞行试验台上可以采取一定的技术措施，如进气加热等，可以扩大模拟的试验包线。  
(6) 机载系统是现代飞机研制和试飞的重头戏，门类多、技术更新快。为了保证型号试飞安全，提高试飞效率，机载系统试飞验证平台不但必要，而且数量较多。美国波音757（FTB）航空电子空中实验室的试飞验证伴随了F-22从方案论证到整机定型的全过程；美国全电飞机技术在F-35上的应用是经过了20多年和多项试验机验证计划取得的成果。当前网络体系作战的新模式给试飞验证带来了新的挑战，利用多机种平台，多试飞场地和设备进行多机协同攻防信息网和数据链验证试飞是一条现实的途径。

# 附录I 典型的试飞员驾驶技术

试飞员驾驶技术不是一个单纯的操纵动作，它是一套从理论到方法的应用体系。其基本要素是：理解任务的目的，掌握试飞方法，熟悉任务成功的判据，了解影响任务的因素，把握试飞安全尺度。下面以战斗机试飞为例，以性能和飞行品质为基础，列举一些典型的试飞驾驶技术，在这个基础上，试飞员们可以根据自己任务需要增加和完善更多的内容。但作为一个合格的战斗机试飞员，下面列举的这些项目都是必须掌握的基本功。

# 1 滑行试验

# 1.1 滑行试验目的

——飞机地面操纵特性。

——飞机地面滑行性能。

——外界条件影响，如风的影响、跑道粗糙度的影响、气候影响等。

——初步检查系统使用性和性能。

——检查低、中、高速滑跑特性，包括：

- 速度控制；  
- 前轮控制；  
- 刹车控制；  
- 乘坐品质（振动等）；  
- 转弯灵活性和稳定性；  
- 起落架载荷。

# 1.2 滑行试验方法

——慢滑。  
——刹车。  
——前轮操纵，刹车。  
——直线滑行：

- 迅速偏离截获；  
- 精确保持直线，试验 PIO；  
- 风速风向影响。

——转弯操纵。

——加速滑行（ $150\mathrm{km / h}$ ）：

- 高速滑行；  
减速刹车。

——滑行到 $220\mathrm{km / h}$ ，体会：

- 跑道特性；  
- 纵向操纵效率；  
- 横向特性；  
- 偏航特性；  
- 起落架特性；  
- 方向舵贡献；  
- 推力对俯仰力矩 $C_{m}$ 影响；  
- 初步空速指示。

——抬前轮和落前轮速度：

对抬前轮特性、抬前轮杆力和位移、PIO、轮载开关、前轮刹车、防滑和阻力伞等项目检查。

# 1.3 操纵品质任务和判据

在滑行中同样要进行飞行品质测试和评定，评定判据是：

——迅速截获滑跑直线，没有超调。  
——直线保持精度：对慢速和快速滑行、加速/减速/刹车条件下，偏离 $\pm 1\mathrm{m}$ 。  
——保持跑道方向偏离 $\pm 2^{\circ}$ 和 $\pm 1 / 8$ 跑道宽度。  
前轮操纵：按PIO评定标准。

# 1.4 可变条件

——飞机重量和重心。  
——外挂载荷和构型。  
——干跑道、湿跑道、积雪和结冰跑道。  
——无风和最大侧风。

# 1.5 安全考虑

——系统动态。  
轮胎和起落架承载。  
——速度对侧向过载 $n_{\gamma}$ 的影响。  
——PIO趋势：

- 前轮和偏航操纵；  
- 抬前轮；  
- 刹车、防滑操纵，热刹车。

# 2 起飞试验

# 2.1 试飞目的

——试验起飞特性、起飞操纵和性能。

外部因素影响，包括跑道表面特性、风的影响、温度影响等。

——从松刹车开始，推油门直到收起落架，检查：

- 加速性；  
- 偏航、横向和纵向操纵；  
- 轮载开关对系统的影响；  
- 飞控系统瞬态；  
- 收起落架，CAT-C到CAT-A的转换。

——试验结果：

- 控制律和程序；  
$\bullet$ 起飞性能；  
- 系统工作；  
- 地面效应。

# 2.2 试验方法

——控制推力和纵向加速度 $n_x$ ，刹车。  
——用方向舵和副翼进行航向操纵。

——抬前轮，注意：

- 速度可预测性；  
建立希望的俯仰角；  
- 对杆力、杆位移的响应；  
- 飞机俯仰响应和稳定性。

——升空，注意：

- 速度和迎角/俯仰角；  
- 俯仰控制；  
- 杆力、杆位移随飞机重量和地面效应的影响。

——加速：

- 起落架收起，瞬态响应；  
- 推力改变。

地面监控参数：

$V, n_{x}, P, Q, R, \mathrm{LGH}$ （起落架），WOW（轮载）等。

# 2.3 飞行品质试验和评定判据

滑跑特性：

- 偏离跑道角： $\pm 2^{\circ}$ ；  
- 偏离跑道：±1/8跑道宽。

——抬前轮：

与预测的俯仰角偏差 $\Delta \theta = \pm 2^{\circ}$   
- 俯仰角保持误差 $\Delta \theta = \pm 1^{\circ}$   
- PIO评定等级。

——升空与加速： $\Delta \theta = \pm 1^{\circ}$   
——起落架收起/推力改变：不引起俯仰改变。  
对每种构型试验和评定次数：3。

# 2.4 改变条件

——推力：军用 $\rightarrow$ 最大。  
——气象：高低温、风速风向、下雪天。  
——飞机重量与构型。  
——各种跑道面特性，如粗糙度、干和湿、雪和冰。  
场高。

# 2.5 安全考虑

——加速判据。  
——起飞中断程序和决策。  
——俯仰几何尺寸限制（如擦地角等）。

- 飞机性能；

# 附录Ⅱ 试飞员人为因素及其防范

许多关于飞行事故报告和文章都得出这样的结论：所有飞行事故原因中有三分之二是人为因素；这三分之二的人为因素中驾驶员的因素又占三分之二。这个结论未免有些武断，因为它并没有进一步说明造成驾驶员犯错误的因素是什么？哪些是主观的，哪些是客观的（如工程上的和管理上的）。

试飞员也是人，是人就有疏忽，就有出错的时候，这种疏忽和出错有时会带来巨大的损失，包括生命、财产和政治损失。同时我们也应看到，试飞员不是普通的人，他们肩上的巨大责任，对他们严格的选拔和培训就是要求他们尽可能不犯或者少犯错误。下面对飞行中可能出现的人为因素及防范措施作一简要描述，便于在安全的管理工作中加以检查。

# 1 人的感知因素

# 1.1 感知方面可能存在的问题

——对距离、间距、高度和速度的误判。

——视觉方面出现问题，包括：

- 无明显特征的地形，如沙漠、水面、干湖床等；  
- 天空昏暗；  
- 前方呈现黑洞；  
- 没有或者是假的地平线；  
- 山区地貌或带斜坡的跑道；  
直升机旋翼下洗气流影响；  
- 异常的灯光干扰；  
- 正对或背对阳光。

——前庭感觉方面可能存在的问题：

- 复合旋转对前庭强刺激造成晕眩感觉；  
- 过载引起对上仰和下俯的错觉；  
- 复杂运动引起的对角旋转方向的错觉；  
- 大过载引起的对滚转角的错觉。

——空间感丧失和晕头转向，包括三种形式：

- 失去方向感，而自己还不知道；  
- 头脑晕眩，不能自治；  
- 前庭和视觉交叉耦合失去判别能力。

——引起对身体位置和飞机姿态错误感觉的因素：

- 失去目视和飞机姿态的参考；

- 加速度造成的错觉；  
- 不利的身体状况：药物、脱水、疲劳等。

一对情景失去感知：

- 地理转向，偏离航线、偏离地图、迷路；  
- 完全不知自己的处境，对危险到来毫无察觉；  
- 错误地判断所处状况；  
- 不能随机应变，束手无策，一错到底；  
- 不能预测和预感发展趋势；  
- 错误假设，主观臆断；  
- 不能确认偏离；  
- 负担过重，时间紧迫。

——注意力分散：

- 飞行检查漏项，被其他事分心；  
- 飞行过程中懒于监控和体察，不注意扫视座舱显示；  
- 对通话和警示无动于衷；  
- 操纵错误，包括忘掉设置、不经意操作、操纵错位、张冠李戴、操纵反向、操纵不到位和不准确等。

# 1.2 影响注意和情景感知的原因

——精神负荷重，解决问题牵涉精力大，或者正处理的信息量过大。

——习惯势力影响，以往飞老飞机的经验和习惯难以适应新飞机飞行。

——不专心。  
——专注一点，不及其余，或任务单一或单调。  
——内、外因素引起的分心。  
——任务负荷大，包括任务压力和时间压力。  
精神紧张或疲劳过度。  
——作战任务工作负荷重。  
——下任务讲解讨论不够，试飞准备不充分。  
——培训不够，经验不足。  
——新机培训中不正确的知识授受。  
——不利的气象条件。  
——不利的座舱环境。  
——战术环境和显示信息过于复杂。  
——驾驶员动力不足，缺乏热情，重视程度不够。  
——差的座舱设计，布局不合理。

# 1.3 针对感知方面的风险管理

对驾驶员进行不断的安全教育和培训。  
——与航空医学机构和飞行安全管理密切合作，按要求进行检查和培训。

——严格控制放飞气象条件。  
——使用任务分析矩阵，进行风险分析。  
——建立驾驶员规范，进行高风险任务管理，如夜航、气象、疲劳、培训等都应有相应的规定。

——过分激情冲动。  
——不可控的生存压力。  
——轻率和优柔寡断的内在性格。  
——糟糕的指挥氛围和安全文化。

# 5.3 判断和决策的风险管理

——加强制定和贯彻可飞和不可飞（Go/No Go）的判据。  
——通过实例讲解如何判别风险。  
——对驾驶员进行判别培训和评估。  
——监督和纠正冒险行为。  
——建立人员管理规范。

# 6 试飞员协调

# 6.1 协调方面的问题

——没有充分的任务计划，下任务讲解或飞行协调准备不充分。  
——没有良好的交流计划，或意图没有得到沟通。  
没有建立标准的语言。

——听不懂、看不清、误解。  
——相互提问、相互检查和监督不够。  
——相关部门没有给出关键的关于安全的数据。  
——疏于领导和组织协调。  
没有应用所有有用资源。  
——人员之间矛盾。

# 6.2 协调不善的原因

缺乏交流和协调的相关训练。  
——没有有效的驾驶员资源管理规范。  
——领导对驾驶员协调的要求和支持不够。  
——驾驶员的资历和经验没有拉开档次，形成梯队，造成互不服气。  
——领导安全文化对驾驶员安全观没有积极影响。

# 6.3 驾驶员协调的风险管理

——建立协调交流规范。  
——进行协调沟通训练。  
——明确试飞队伍的岗位分工。  
——下任务讲解，飞行后讲评，飞行中的交流沟通。  
——通过专门培训进行驾驶员协调管理任务和技术的教学观摩课，提高驾驶员的素质。

# 7 飞机设计和系统问题

# 7.1 设计和系统中影响安全的因素

——错误的开关、摇臂和操纵。  
——容易混淆的座舱显示。  
——座舱操纵不易看、不易达。   
——对座舱告警的处置方法不明确，甚至误处理。  
——模态混淆。  
——影响驾驶员精力分配。

# 7.2 设计造成安全的原因

——不良的主飞控方案。  
——主要数据显示或显示格式不合适。  
——座舱显示数据大小、方位、颜色、闪烁和亮度不合适，读数困难。  
——座舱显示控制或系统响应不协调。  
——没有必需的危险提示或告警，或者提示和告警级别分配不当。  
——糟糕的人机显示界面和适用性。

国

——糟糕的飞机和系统说明书。  
——没有足够的系统支持或设施，如飞行模拟、导航、机动和交通管制支持设施等。  
非标准的座舱布局。  
——不适当的自动化水平，或过于复杂的操纵模式。

# 7.3 设计和系统风险管理中

——对试飞员加强飞机与系统培训，并参加试验。  
——提供充分的、准确的技术资料，通过读书飞行（Fly by the Book）。  
——改进试飞中出现的问题，来不及改进的也要采取临时措施。  
——建立足够的标准，充分的性能预测。  
——足够的资源支持试飞任务完成。

# 8 领导因素

# 8.1 领导方面的问题

——不合理的程序计划和驾驶员任务分配。  
——不注意监督驾驶员的职责和休息。  
——没建立足够的标准。  
——没监督标准贯彻情况。  
——没重视驾驶员的资源和培训。  
——没控制和调离高冒险型驾驶员。  
——没建立和监督质量标准。  
——故意违反，或导致他人违反标准、规划和规定。  
——没有监督、调研和评定实际任务风险，包括：

- 环境危害和使用状态；  
- 任务要求和驾驶员技术水平；  
- 驾驶员和系统限制（飞机、维修、机场、交通管制、飞行手册等）。

# 8.2 导致领导问题的原因

——任务多、速度紧，组织管理工作负荷过重。  
没有良好的安全文化和氛围，松散的管理。  
——没有通畅的上下沟通渠道。  
一对高冒险驾驶员没有控制环节。  
——没有足够航空医学人素工程培训。  
没有航空医学标准。

# 8.3 管理层的风险管理

——建立良好的安全文化和氛围。

——管理者尽到安全管理的责任。  
——把安全作为本单位的目标。  
——建立明确的执行标准。  
一对驾驶员进行充分的培训。  
——监督驾驶员规范执行。  
——了解自己和自己的团队。  
——开放的思路和开放的政策。  
——纠正执行不力和违规。  
——领导抓典型，好的或差的典型，奖励安全行为。  
——强化风险管理。  
——科学、客观、公正的事故分析。  
——了解飞机和系统限制和容差。  
——了解座舱布局的不合理性，了解控制和显示的设计缺陷。  
——公开危害信息，建议设计更改飞行使用手册。  
——关注机场、交通管制、制造商（飞机、发动机、航电等制造商）和合同商等相关信息。
