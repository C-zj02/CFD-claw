# 第12章 机体轴承

# 12.1 概述

本章所述的机体轴承是指飞机机体结构使用的轴承，主要是指飞机操纵系统和操纵面使用的轴承，而不包括特种设备和机构中使用的轴承，如发动机、发电机、减速器、仪表和起落架上使用的轴承。

机体轴承按几何形状、运动方式和使用性能可分为三类：减磨轴承、球面滑动轴承和杆端轴承。减磨轴承又按不同的滚动体分为球轴承、滚子轴承和滚针轴承；球面滑动轴承我国习惯上称关节轴承，按润滑方式分为可再润滑轴承和自润滑轴承；杆端轴承是推拉杆端部连接用的轴承，其轴承座与杆体构成一个整体，可装减磨轴承和关节轴承，构成杆端球轴承、杆端滚子轴承和杆端关节轴承。

# 12.3 轴承的安装与固定

轴承是机械元件，用它将载荷从一个结构件传递到另一个结构件，且同时允许两结构件有相对运动。轴承的不正确安装与固定可导致使用寿命降低，甚至造成壳体或轴承元件的损坏，因此，正确的安装与固定工艺和技术，是保证轴承正常的工作性能所必需的。

轴承的安装与固定可参考美国军用标准 MIL-STD-1599 第 200 的 202 条要求：轴承的固定。

# 12.5 机体轴承国际标准

国际标准化组织滚动轴承委员会（ISO/TC4）和航空航天飞行器委员会（ISO/TC20）组成一个机体轴承联合工作组，负责制定航空航天机体轴承国际标准，下面是已发布的一套航空航天机体滚动轴承标准。

该类轴承预定用于飞机结构和其操纵面上。允许使用温度为 $-54\sim 150^{\circ}\mathrm{C}$ 。技术条件按ISO14190“航空航天机体球轴承和滚子轴承技术条件”。

# 附录 三维数字产品定义与并行工程简述

本节内容取自波音公司波音767-X三维数字产品定义（B767-X3D DIGITAL PRODUCT DEFINITION），作为三维数字产品定义设计方法与程序的简要介绍。

计算机技术的发展与应用使飞机设计进入了一个崭新的阶段，即飞机的零件在还没有生产之前，可以在计算机上模拟飞机的装配与协调。从而减少了设计错误，降低了设计与制造的成本，并保证了飞机产品的质量。这种新的设计概念称之为三维数字产品定义与并行工程。

三维数字产品定义是飞机研制过程中几何图形部分设计的首选方法。在对几何图形进行更改中，在资源与时间上会产生直接效益，并对进一步的自动化设计产生长远的潜在效益。

设计人员应使用CATIA CAD/CAM系统定义所有的飞机零件、数据预装配和工艺装备设计。在设计工作中使用综合数据管理系统（IDM）发放零件数据集、使用导线信息发放系统（WIRS）发放导线数据、使用综合工作说明（IWS）程序定义设计方案、图树和材料表（BOM），并使用综合计划系统计划工程活动。

# F1 三维数字产品定义的内容

三维数字产品定义包括下述三方面的工作。

# （1） $100\%$ 数字产品定义

飞机零件的数字定义即使用计算机辅助设计与制造系统CATIA（Computer-Aided Three-Dimensional Interactive Application）建立三维的零件模型。CATIA可将三维模型转变为二维图样，飞机零件模型是三维实体元素。设计人员可在计算机上方便地进行零件预装配，校核零件的干涉与界面的配合，精确进行重量、平衡、应力分析。设计制造人员可较早看到并了解零件的实际形状，并可以从实体上提取剖视图，所取得的数字数据可用于数控加工。产品的安装更方便更精确。客户服务部门可使用CAD数据产生技术出版物。

所有的零件设计数据是惟一的批准的数据集，为下游的使用者使用，并且更改仅对数据集而不是图样。每个零件的数据集包含一个三维模块和一个二维图样。

# (2）3D实体数字化预装配

数字化预装配是在计算机上建模和装配零件的过程，以便在加工前验证配合与界面的正确性。数字化预装配的成功与否取决于与其他人分享设计得出的结果。数字化预装配可以降低因设计错误引起的更改与重复工作。数字化预装配的目的在于设计发放前消除干涉。设计人员能够访问其他设计人员的设计，验证是否正确并达到设计一体化。

其他专业，如工程分析、材料、计划、工具、客户支援等也可以访问设计结果，并在发放前向设计提供反馈意见。

数字预装配也用于结构与系统设计、管路线路的布置和某些安装的演示与零件拆除。

# (3) 并行产品定义

并行产品定义是对产品与其相关工艺同时进行设计的系统化途径，包括工程设计、制造

和产品支援等方面。

这一系统化途径目的是使研究人员从初始概念到最后交付产品的整个寿命周期内考虑所有的因素，其中包括质量、成本、计划和使用者的要求，等等。

# F2 三维数字产品定义的工作基础

为在飞机设计中实现三维数字产品定义与并行工程，应进行下述相应的工作：

# （1）组建设计工作队

建立由来自工程设计、制造、供应商/参加方/材料、财务和计算机系统的关键人员组成的设计工作队，共同研究具有生产性的设计，使设计更改、错误和重复工作减至最少。

# (2）数字定义工具

工具的数字定义是工具制造的惟一的根据。对工具的设计、制造和验证要 $100\%$ 使用数字定义来确定工具零件数据集。

(3) 定义零件一工具、工具一工具之间关系

使用3D数字数据集，模拟零件与工具的实体模型，验证配合、功能和装配能力。

# (4）增加制造数据

制造要对设计制造条件下的安装件、装配件和零件提供制造后的构型。

制造要对发放数据集模型标识等轴分解视图，以便对要完成的工作提供图示说明。制造还要标识数据集的关键特性，以支持制造产品的偏离控制。

# （5）用数字数据图表示计划和生产说明

包括图形标识的文字制造计划应能更好地交流要完成的工作。应以三维几何图形研究图解计划，并用打印生产图示说明来代替手工绘制。

# (6）工具的协调

计算机收集系统应储存工具一工具及工具一零件关系的数据。保证工具一工具及工具一零件之间的协调。

# (7）综合计划

制定综合计划，使其成为一种工具，用以改进通信与涉及飞机设计、计划、制造、试验与交付计划的功能。

# （8）综合工作说明

综合工作说明包括设计工程数据（工作说明和材料表）、制造工程数据、工具工程数据（工作说明和材料表）、经费数量和某些计划。

综合工作说明要在产品研制的早期开始，并在整个产品的研制过程中不断改进。

综合工作说明要以文件的形式产生，与产品定义同时进行，以避免产品发放过程中产生意外的错误。

# (9）自动差异控制

建立自动差异系统，是通过电子传输、目视、跟踪、显示、使用期限和管理的协调，缩短设计问题处理的时间。电子拒签，在解决设计问题的过程中，在计算机系统中应保持有效的状态。

# (10) 材料表的改进

在设计人员控制下，用数字输入代替手写的自动零件目录输入，减少图纸与材料表之间

的匹配错误。用工程标准分发系统代替标准和程序文件。

（11）使用CATIA处理地面支援设备与技术出版物

利用数字化预装配，以三维实体设计以及模块化处理地面支援设备。技术出版物直接使用从工程数据传输来的数据。

# F3 三维数字产品定义的设计工作队

为进行飞机设计的三维数字产品定义工作，应建立必要的设计工作组，井包括所需要的技术专业，各专业开展协调有序的工作。

（1）在设计工作队中，来自各专业的设计人员共同工作，设计并支持选定的型号。产品定义工作一体化使设计更改、错误和重复工作达到最少。来自不同专业的人员在一个工作队环境里并行进行工作实现一体化。在产品数据集发放前，在产品定义方面合并下游组织的更改建议，更进一步减少设计更改、错误与重复的工作。  
(2）完成设计工作的主要途径，是设计工作队在工程、使用、材料、客户服务和其他方面共同合作的原则下进行工作。为达到最佳的设计目标，设计工作队必须把高水平、可生产性和可维护性的技术输入给型号设计。  
(3) 对工程和制造部门，要研制出更具生产性、错误更少的设计。工程部门负责设计，并且在设计中最大限度地综合制造部门的可生产性技术。更精确地定义零件与装配件中使用的数字产品数据，在设计中并行地定义系统和结构。工程设计部门使用数字产品预装配，确认所有零件已经被定义并且互相配合。制造部门通过生产计划，完成工具设计验证与可生产性，并对工程设计提供有效的技术支持。

# F4 设计研制的过程

设计研制过程是一个迭代循环过程，从建立3D模型开始，对3D模型进行数字化预装配、修改并共享设计，直至设计完成，所有零件相互配合。最后用设计模块形成零件、装配件和安装件，完成2D图样并予以发放。在设计研制过程的各阶段，设计工作队协调设计工作。

（1）对飞机零件建立3D数字产品定义，这些设计是在飞机坐标系建立模型，设计完成时，这些设计发展成为零件、装配件和安装件。  
(2）用数字化预装配共享文件，分配3D实体、装配件和安装件。每个设计人员必须在设计或安装件上不断对数字化预装配分配最好的设计方案。如果模型的大小受到限制，设计人员必须对数字化预装配分配子安装件或装配件。每个分配的模型必须在飞机坐标系确定准确的位置（沿飞行方向， $X$ 轴向后为正， $Y$ 轴向右为正， $Z$ 轴向上为正）。  
(3）设计零件时，不断对零件所处位置的数字化预装配进行搜索。每个设计人员必须经常搜索装配件所在区域的数字化预装配，以支持设计的一体化。  
(4）建立设计零件和配置零件的数字化预装配，用于校核干涉、界面匹配和系统的路线。经常搜索并重新建立数字化预装配，保证已考虑到最新的零件构型。  
(5）对配置的零件要经常进行干涉校核，设计人员要召集其他设计人员和制造人员解决干涉或其他的设计与生产问题。如果需要，应把干涉模型放到联机活动项目文件夹。

(6) 使用 CATIA 数据分析零件，并提供反馈意见给设计人员。分析人员召集有关的设计人员解决干涉或其他的设计与生产问题。并把有问题的模型放到工程反馈文件夹，或使用打印校核文件。  
（7）审查关于制造的数字化预装配，并尽早反馈有关生产性的信息。计划人员和工具设计人员要召集合适的设计人员和制造工程师解决干涉与生产性问题，并把有干涉的模型或问题放到制造反馈文件夹。  
（8）检查未解决的问题、设计分析与制造反馈文件夹。  
(9) 解决工程分析与制造要求的干涉与界面问题，并进行设计更改。联机反馈过程只是对信息交换的补充，并不代替电话与个人的信息交换与协调。在使用联机文件夹之前，与个别人接触解决出现的问题。如果问题得不到解决，把模型放到联机反馈文件夹。  
（10）不断重新分配预装配共享文件夹，保持预装配共享文件夹存有最精确的设计数据。  
（11）不断迭代上述过程，直至设计完成。这一循环过程继续到所有零件配合，并确信没有干涉为止。不要等到设计发放再校核干涉和界面的配合。  
（12）完成模块的2D图样视图使之成为成熟的设计，设计人员在零件的2D图样视图上添加尺寸、附注，并标识零件的关键特性。  
(13) 冻结数据集。  
（14）如果需要，按制造部门反馈的信息修改数据集。  
（15）发放数据集。

# F5 数字化预装配

# F5.1 数字化预装配的基本内容

数字化预装配是三维数字产品定义的重要组成部分。

（1）数字化预装配是利用模型计算机文件的过程，这些计算机文件包含零件、装配件和安装件的设计，能按设计、分析、计划或工具人员的需要予以装配。  
(2) 数字化预装配是在计算机上对飞机的装配进行模拟。以3D实体元素建立零件模型，并检查干涉、界面配合和设计一体化等各方面工作。数字化预装配过程能够识别并解决整个飞机的所有的干涉问题。  
(3) 对整个飞机或飞机的部件的数字预装配，模块的大小对反应速度是一个重要的影响因素。必要时应对参与预装配的部件进行分解，满足反应速度的需要。  
(4) 数字化预装配使用 CAD/CAM 系统，在计算机系统的空间里，完成数字数据集模型装配所要求的实体模型。

# F5.2 数字化预装配的主要步骤

建立数字化预装配包括七个主要步骤：

（1）建立。使用3D元素或实体元素在三维空间建立设计或安装模型。  
(2）共享。使用COMMENT软件输入有效的位置数据，进入3D零件，共享模型。  
(3) 管理。管理预装配数据，预装配管理人员检查模型的内容、格式和特征数据，模型原本的复制件存入共享文件夹，以便工作组了解建立模型的历史情况。

(4) 搜索。在给定区域搜索所有的零件, 每个设计人员使用搜索软件和平面文件数据库, 发现配置的零件。  
（5）校核。对干涉进行检查，每个设计人员建立数字化预装配，并检查干涉、界面的匹配、孔的对中和飞机部件总体区域的安排。  
(6) 解决。解决所有的设计问题，设计人员交换并协调所有问题或相关问题的结果。联机反馈文件夹放置有问题的模型。  
(7) 再共享。再次共享数字预装配的设计，再次反复共享修改的模块。这一循环继续到完成所有的零件设计，并达到所有的零件完全配合。

除数字化预装配之外，设计人员要合并来自工程分析、试验和制造方面的反馈意见。数字化预装配模块的数字管理是一项巨大而繁重的工作，由数字化预装配管理部门进行。这一外部组织机构管理数字化预装配模型，保证所有的使用者容易进入，并在发放时进行最终的检查。

# F5.3 数字模型的定义

数字模型和数字化预装配具有相同的意义。数字模型和数字化预装配可以分为三个阶段。总的来说应尽可能精确地建立零件的模型。

（1）数字化预装配的第一阶段是空间的定义，足够详细地建立基本形状、包围的空间、工程设计组之间协调空间的分配。这些用来支持“初步结构/系统界面项目”进展计划。  
(2) 数字化预装配的第二阶段在生产数据集发放之前，工程设计部门使用数字化预装配研究和验证设计构型。数字化预装配用于硬件设计和设计组之间的界面协调，并且支持设计评审。零件的形状已经确定，但还没有详细的设计特性。在本阶段数字化预装配已进展到足以用可靠性、价值工程因素需求、地面支援设备的兼容性来检查飞机设计的可接近性、维护性和使用性，但没有详细到固定和最后安装的要求。制造部门对飞机构型已熟悉，工具设计与生产顺序计划容易进行。数字化预装配的第二阶段支持“确定系统走向与主要固定支撑”工作计划的完成。  
(3) 数字预装配的第三阶段是完成详细设计零件的预装配，表示飞机已经研制到管路、导线束、控制电缆、绝缘层、空调导管，等等，如有关电路、液压管路、夹子、角片、紧固件和安装孔的最终的加工与安装。在发放工程数据之前，完成数字化预装配，并且确保零件之间不存在干涉问题。

# F5.4 并行产品定义

并行产品定义是对包括制造和产品支援的产品设计一体化，和与其相关过程并行设计的一种系统方法。这种方法要使研究人员从开始通过控制的概念考虑产品寿命周期内所有的因素，包括：质量、成本、计划和使用者的要求。

并行产品定义是在设计工作队内模拟设计、分析和相关飞机零件的计划和产品支援工具的设计，并建立这些零件数控程序。

并行产品定义包括下述方面：

(1) 结构系统设计；  
(2）分析产品支援的设计发放数据；  
(3) 较早地、连续地从所有的设计工作队人员得到反馈信息；

(4) 完成发放时零件、装配件和安装件的关键件计划；  
(5) 在零件发放时完成主要工具设计；  
(6) 发放前数控计划与编程;  
（7）发放前供应商与附件的问题；  
(8) 飞机制造用几何图样与材料表的批准，产品定义的发放；  
(9）地面支援设备、取证文件、功能试验规范和用户技术出版物的并行研究。

# F6 设计细节

# F6.1 结构设计

结构设计包括下述内容：

（1）结构设计人员对总的结构设计过程负责，并在指定飞机部件内负责整个区域的管理和设计一体化，结构设计人员为每个零件建立3D/2DCATIA模型；  
(2) 建立3D实体零件几何图形，支持数字化预装配、零件—零件和零件—工具设计一体化、下游工作的目视化需求、重量分析、生产说明、维护通道的验证和技术出版物等项工作；  
(3）结构设计人员负责实现设计的预定功能，保证结构零件无设计错误，验证设计配合、界面和对其他所有的附带零件不存在干涉。所有的系统规定，包括紧固件和其安装位置、支架、结合面等应在结构设计中发出；  
（4）结构设计人员解决所有的干涉问题，考虑所有的系统管路敷设问题，并用数字化预装配检查所有的间隙，在必要时，为避免生产出现问题，必须包括紧固件并检查干涉。

# F6.2 绝缘层的设计

绝缘层的设计包括下述内容：

（1）绝缘层的设计组负责绝缘层总的设计过程和数字化预装配，绝缘层应用CATIA设计而不是用实体模型。这样大大减少了实体模型的需要，并把绝缘层的设计责任放给了所属的设计人员；  
（2）对飞机内部进行了防寒、防湿、防噪声的绝缘层设计要根据机体内部形成各种各样的形状。

# F6.3 驾驶舱的设计

驾驶舱设计包括下述内容：

(1) 设计人员负责仪表板和驾驶员周围环境的 3D 设计，包括遮光板、座椅、驾驶员飞行控制界面等；  
(2）设计人员提供为数字化预装配共享的设计、安装与装配；  
(3) 驾驶舱设计人员对驾驶舱的整个设计过程和预装配负责。此外，驾驶舱设计人员使用人体模型软件进行目视与可达性研究。用3D实体数据集，规范控制零件，如：航空电子黑匣子，建立实体模型用于数字化预装配，并进行发放与维护，静压管设计与发放在液压部分进行说明。

# F6.4 推进系统

推进系统设计人员负责所有结构与推进系统零件的配合，并负责推进系统的整个设计过程与预装配。管路的建立与发放在液压部分进行说明。

# F6.5 飞行控制

飞行控制设计包括下述内容：

(1) 飞行控制设计人员负责驾驶舱控制、活动面、自动驾驶、调整面、推力与操纵控制，包括所有上述硬件的安装；  
(2）飞行控制设计人员负责与结构和系统连接的零件的配合，并负责飞行控制系统的整个设计过程与预装配；  
(3) 机械系统设计人员建立 $100\%$ 的 3D/2D 数字产品定义。建立 3D 实体零件几何图样, 支持数字化预装配、零件一零件和零件一工具的设计一体化和下游工作的可视化需求与重量分析。这样可以改进零件设计, 减少更改, 减少实体模型, 减少设计流程的时间, 并提高零件的制造性。

# F6.6 空调与冷却系统导管设计

空调与冷却系统导管设计包括下述内容：

（1）系统设计人员负责空调系统和电气/电子冷却系统的3D设计，包括所有导管以及空调组件的安装；  
(2）系统设计人员负责所有与结构和系统连接的零件的配合，并负责系统的整个设计过程与预装配；  
(3) 以 CATIA 设计所有的导管, 使用导管软件从数字定义中提取柔性导管的长度。系统与结构的并行设计使导管设计人员能够使用更多的标准弯曲参数。

# F6.7 液压管路设计

液压管路设计包括下述内容：

（1）系统设计人员负责所有与结构和系统连接的零件的配合，并负责液压管路设计的整个设计过程与预装配；  
(2) 系统设计人员，以结构为背景使用实体数字预装配，用CATIA在三维空间确定液压管路的路线。使用管路软件提取管路装配数控数据集，定义交换格式驱动数控弯曲机。这样可以相对于周围的结构与系统，进行并行的管路一体化研究，在发放前保证正确的管路形式、配合与功能，并验证安装点与敷设情况。

# F6.8 标准件图形库

工程标准分配系统储存基本的标准特性，典型的标准件包括：紧固件、垫圈、垫片、轴承、导线接头、液压系统接头、管夹与制造工艺信息。这些标准图形用于减少设计的重复性。

# F6.9 设计零件图形库

设计零件图形库可使设计人员从现有的工程项目迅速方便进入频繁使用的零件，例如：导管夹、角片与衬垫等。CATIA的设计零件图形库应与标准件图形库互相兼容。

# F6.10 电子/电气电缆的布置

电子/电气电缆的布置设计包括下述内容：

（1）系统设计人员负责所有与结构和系统连接的零件的配合，并负责电子/电气电缆的布置的整个设计与预装配；  
(2) 系统设计人员使用系统设计、原理图和导线综合数据在 CATIA 上确定 3D 导线束的路线，并以确定的导线束为背景进行数字化预装配。导线束的长度、设备信息和相关的导线束数据可从 CATIA 提取。工程发放包括导线数据与 3D CATIA 布线。

# F6.11 电子/电气硬件

所有的电子/电气硬件以CATIA进行设计与发放，包括：支架、接头、照明、仪表和线路板。设计人员负责电子/电气硬件的整个设计与预装配。
