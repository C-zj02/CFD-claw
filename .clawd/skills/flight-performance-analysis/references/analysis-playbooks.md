# 飞行性能分析剧本

本文件用于完整或复杂性能分析。轻量问答只需读取相关小节。

## 1. 意图到流程映射

| 用户意图 | 必要输入 | 推荐流程 | 典型输出 |
| --- | --- | --- | --- |
| 整理气动/动力数据 | 数据文件、单位、构型、适用范围 | 数据清洗 -> 单位统一 -> 状态标注 -> 缺口清单 | 特性数据 CSV、数据质量报告 |
| 选取或调整剖面 | 飞机类型、任务目标、速度高度、航段约束 | 剖面模板 -> 航段参数 -> 重量递推 -> 确认 | 剖面表、航段图、待确认项 |
| 重量状态管理 | 空机、载荷、燃油/能量、备用要求 | 初始重量 -> 航段消耗 -> 着陆重量 -> 包线检查 | 重量状态表、燃油/能量裕度 |
| 单项性能计算 | 对应模块输入、环境、构型 | 模型选择 -> 输入确认 -> 计算 -> 裕度解释 | 起飞/着陆/爬升/航程等指标 |
| 完整任务性能分析 | 气动、动力、重量、剖面、约束 | 数据校核 -> 剖面 -> 重量 -> 性能 -> 图表 -> 归档 | 汇总报告、结果表、图表 |
| 剖面优化 | 目标、变量、边界、不可违反约束 | 基准剖面 -> 变量定义 -> 约束检查 -> 搜索/优化 -> 敏感性 | 推荐剖面、折中解释 |
| 飞行包线绘制 | CLmax、推力/功率、结构限制、q 限制 | 边界定义 -> 采样 -> 包络合成 -> 图表 | V-n、H-M、H-V 或 q/Mach 包线 |

## 2. 数据质量检查

计算前检查：

- 单位：速度、重量/质量、推力/功率、燃油流量、距离和高度必须统一。
- 构型：起飞、着陆、巡航、襟翼、起落架、外挂、结冰或损伤状态不能混用。
- 适用范围：气动极曲线和动力特性必须记录 Mach、高度、雷诺数或油门范围。
- 插值外推：插值方法可用于计算，外推必须标注风险并请用户确认。
- 重量闭合：航段消耗后的重量不能小于空机加有效载荷加不可用燃油/能量。
- 约束闭合：跑道、失速、过载、动压、升限、发动机包线和运行限制要同时检查。

## 3. 常见性能模块

### 起飞与着陆

关注输入：`W/S`、`T/W` 或 `W/P`、`CLmax_takeoff`、`CLmax_landing`、跑道、机场高度、温度、风、地面摩擦、越障要求。

输出：起飞距离、着陆距离、失速速度、爬升梯度、翼载上限、推重比/功率载荷要求。

确认重点：构型、CLmax、跑道条件、是否含安全系数。

### 爬升与升限

关注输入：阻力极曲线、推力/功率随高度和速度修正、重量状态、爬升速度、爬升率或梯度要求。

输出：爬升率、剩余推力/功率、到达高度时间、实用升限、绝对升限。

确认重点：爬升速度计划、发动机额定状态、温度偏差。

### 巡航、航程与航时

关注输入：巡航高度、速度或 Mach、升阻比、SFC/能耗、初末重量、备用燃油/能量。

输出：航程、航时、燃油/能量消耗、最佳巡航速度/高度建议。

确认重点：是否用喷气 Breguet、螺旋桨航程公式、电推进能耗模型或逐段积分。

### 机动与加速

关注输入：速度、高度、重量、CLmax、结构过载、推力/功率余量、阻力极曲线。

输出：稳态盘旋过载、转弯半径、转弯率、加速时间、能量高度变化。

确认重点：稳态还是瞬时、是否允许高度损失、结构和失速边界。

### 飞行包线

常见边界：

- 失速边界：由 `CLmax`、重量、密度和速度确定。
- 结构边界：正负过载限制。
- 动压边界：由最大允许动压或气动弹性限制确定。
- 推力/功率边界：由剩余推力/功率为零或爬升率要求确定。
- Mach/温度/发动机边界：由动力装置和气动适用范围确定。

输出图表优先级：`V-n` 用于结构和失速，`H-M` 用于高度 Mach 包线，`H-V` 用于飞行剖面与高度速度限制，`range/endurance vs speed` 用于剖面优化。

## 4. 剖面优化流程

1. 定义基准剖面：航段、状态、速度、高度、重量和构型。
2. 定义目标函数：燃油/能量最小、航程最大、时间最短、爬升时间最短、或多目标折中。
3. 定义变量：巡航高度、速度/Mach、爬升速度、爬升顶点、油门/功率设置、航段距离。
4. 定义约束：失速裕度、最大动压、发动机包线、跑道、升限、过载、温度、备用燃油/能量。
5. 先做粗网格或参数扫描，找出敏感方向；只有输入足够可靠时再做数值优化。
6. 输出推荐剖面，同时列出为什么不是单纯追求最优数值，例如裕度、数据可信度、运行限制或用户偏好。

## 5. 可见分析表达

每个主要回复可以使用：

```text
分析思路摘要：
1. 我判断本次任务属于 <类别>，核心指标是 <指标>。
2. 当前可直接使用的输入有 <输入>；缺口是 <缺口>。
3. 我会先采用 <模型/流程>，并在 <节点> 请你确认。
4. 这一步的输出会保存为 <产物>。
```

计算后使用：

```text
结果解释：
- 主要结果：<数值、单位、条件>。
- 关键驱动因素：<1-3 个因素>。
- 可信度：<已确认输入/假设/外推风险>。
- 下一步：<需要用户确认或更高保真分析的内容>。
```

这种表达体现工程判断和步骤，不展示逐字内部推理。
