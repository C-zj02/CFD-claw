# 气动/进排气评估流程

## 任务分型

将用户需求先归入一个主类型：

- `external_aero`：全机/部件外流气动，输出气动力、力矩、压力/马赫数云图等。
- `intake_duct`：进气道/内外流耦合，输出总压恢复、畸变指数、质量流量、AIP 面分布等。
- `nozzle_exhaust`：排气喷管/尾喷流，输出流量、推力、推力系数、出口均匀性、喷流结构等。
- `coupled_propulsion`：进气道-发动机面-喷管耦合评估，输出进排气匹配、损失和安装影响。
- `postprocess_only`：只整理已有仿真/试验数据，通常用 MATLAB 或 Python 后处理。

## 状态机

1. `draft_case`：解析需求并建立 case 草稿。
2. `awaiting_confirmation_a`：确认任务目标、几何、工具、run_mode 和默认模板。
3. `geometry_gate`：检查几何可读性、单位、边界命名、模型质量。
4. `mesh_gate`：确认网格数量级、y+、边界层、加密区域。
5. `solver_gate`：确认湍流模型、定常/非定常、CFL、迭代、监测量。
6. `ready_for_dry_run`：生成配置并做离线校验。
7. `ready_for_mock`：生成流程演示数据。
8. `ready_for_execute`：外部工具和许可证已确认，可以真实运行。
9. `monitoring`：读取日志、残差和监测量，识别发散、振荡或质量不守恒。
10. `postprocessing`：计算指标、生成云图/曲线/表格。
11. `archived`：写出结果包和摘要报告。

## 几何质量门禁

没有工具时只做文件级和命名级检查；接入工具后再扩展为真实几何检查。

必须检查：

- 文件是否存在，格式是否在允许列表内。
- 单位是否明确，默认不可静默假设。
- 坐标轴定义是否明确，尤其来流方向、重力方向和机体轴。
- 边界命名是否覆盖目标评估所需边界。
- 是否说明了对称面、周期面、发动机面/AIP、喷管入口/喉道/出口。

模型质量未通过时不要直接进入执行；给出修复建议并请求确认。

## 网格策略

快速评估：

- 网格数量级较低，优先看趋势和流程。
- 可采用壁函数，目标 y+ 视模板设置。
- 局部加密只覆盖唇口、AIP、喉道、出口、激波/分离高风险区域。

高精度评估：

- 明确边界层层数、增长率、首层高度或 y+ 目标。
- 加密区域必须可追踪：区域名、类型、目标尺寸、原因。
- 对进气道关注 AIP 截面、弯道、唇口、附面层分离。
- 对喷管关注收缩段、喉道、膨胀段、出口剪切层和激波。

## 求解与监测

常用求解设置：

- 快速定常：RANS，SST k-omega 或 Spalart-Allmaras，先一阶稳定再二阶。
- 高精度定常：RANS/SST，收敛标准更严格，必要时局部网格无关性检查。
- 非定常：URANS/DES/LES 只在用户明确需要或流动本质需要时采用。

监测量：

- 残差：continuity、momentum、energy、turbulence。
- 外流：CL、CD、Cm、侧力/偏航/滚转力矩。
- 进气道：mass_flow、total_pressure_recovery、distortion、AIP 面均匀性。
- 喷管：gross_thrust、mass_flow、thrust_coefficient、exit_pressure_ratio。
- 守恒：入口/出口质量流量差、能量和压力损失。

发散提醒：

- 残差连续上升、监测量跳变、CFL 过高、负体积、质量流量不守恒、出口回流异常都应触发提醒。
- 建议动作包括降低 CFL、改初始化、先一阶、检查边界命名、检查网格质量、放宽后再收紧收敛标准。
