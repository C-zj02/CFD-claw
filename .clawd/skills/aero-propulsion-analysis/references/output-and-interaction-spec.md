# 输出与交互规范

读取时机：需要正式答复、导出文件、补问用户、确认数据来源或形成交付表格时读取。

## 必要确认节点

先确认以下会显著改变结论的事项：

- 精度等级：快速估算、方案论证、详细计算、试验/交付级。
- 数据来源：用户文件、本地 RAG、外部资料或工程假设。
- 符号约定：迎角、侧滑角、舵偏、力矩正方向、参考面积和参考弦长。
- 工况一致性：Mach、高度、构型、舵偏、质量状态、发动机设定。
- 安装推力口径：总推力、净推力、未安装推力、安装后推力、是否含进气道/喷管/短舱损失。
- 导出文件：文件类型、字段、单位、命名、是否覆盖已有文件。

若用户已经明确要求继续或导出，可以直接执行，但仍在结果中标出采用的假设。

## 缺失输入补问格式

```text
我可以继续做快速估算，但以下输入会影响结果：
1. <缺失项>：影响 <指标>，默认可取 <保守假设>。
2. <缺失项>：影响 <指标>，默认可取 <保守假设>。

请确认使用默认假设，或补充这些值。
```

只在缺失项阻断任务时补问。若用户要的是概念方案或范围推荐，可以带假设继续。

## 文件输出规范

默认目录：

```text
outputs/aero-propulsion-analysis/
```

推荐文件：

- `<case>_normalized.csv`：清洗后的输入数据。
- `<case>_results.csv`：特性参数和计算结果。
- `<case>_method.json`：公式、单位、假设、拟合区间和来源记录。
- `<case>_report.md`：中文分析报告。

CSV 字段名必须包含单位。JSON 必须包含 `source`、`assumptions`、`method`、`results`、`limitations`。
