# 测试与验证记录

## 本地启动

网页端启动命令：

```bash
python -m src.cli web --host 127.0.0.1 --port 8081
```

默认访问地址：

```text
http://127.0.0.1:8081
```

如果端口被占用，可以换成：

```bash
python -m src.cli web --host 127.0.0.1 --port 8090
```

## 回归测试

网页端相关测试：

```bash
python -m pytest tests/test_web.py
```

技能和运行时相关测试：

```bash
python -m pytest tests/test_aircraft_conceptual_design_skill.py
```

```bash
python -m py_compile src/web/app.py
```

```bash

```

## 前端人工验收点

| 验收项 | 预期 |
|--------|------|
| 页面语言 | 主要界面文案为中文，主题面向飞行器设计。 |
| 模型显示 | 模型固定显示为 `deepseek-v4-pro`。 |
| 技能默认值 | 默认显示“技能：无”，能力下拉为“不自动使用”。 |
| 能力显示 | 只显示“飞行器设计”，不显示内部技能名。 |
| 内部术语 | 用户可见区域不出现 `aircraft-design-rag`。 |
| 资料能力 | 选择“飞行器设计”后，回答可结合本地资料。 |
| 流式完成 | 后端发送 SSE `done` 后，前端应结束忙碌状态。 |
| 公式渲染 | `$...$`、`$$...$$`、`\(...\)`、`\[...\]` 能显示成公式样式。 |
| 代码保护 | 代码片段中的 `$...$` 不应被当作公式渲染。 |
| 过程记录 | 助手回答显示“处理中/已处理”状态，并可展开查看过程事件。 |

## 已知限制

- 当前会话历史主要保存用户和助手消息，工具过程事件还没有完整持久化。
- 网页端 LaTeX 渲染是轻量实现，主要覆盖常见飞行器设计公式。
- 真实模型响应速度会影响人工验收时间，必要时可用测试服务模拟 SSE 流。
