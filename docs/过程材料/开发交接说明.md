# 飞行器设计功能开发交接说明

## 代码入口

| 模块 | 文件 | 说明 |
|------|------|------|
| 网页端服务 | `src/web/app.py` | 本地 HTTP 服务、HTML/CSS/JS 页面、会话 API、SSE 流式接口。 |
| 本地资料服务 | `src/web/rag_service.py` | 飞行器设计本地资料索引、缓存、检索服务。 |
| 技能定义 | `.clawd/skills/aircraft-conceptual-design/SKILL.md` | 飞行器总体设计技能主提示词和工具策略。 |
| 资料检索技能 | `.clawd/skills/aircraft-design-rag/SKILL.md` | 本地资料检索能力定义。 |
| 检索脚本 | `.clawd/skills/aircraft-design-rag/scripts/search_rag.py` | 从 `RAG-data` 检索 Markdown 资料。 |
| 约束图脚本 | `.clawd/skills/aircraft-conceptual-design/scripts/plot_constraint_boundary.py` | 生成方案界限线图 CSV/SVG。 |
| 网页端测试 | `tests/test_web.py` | 网页端 API、技能映射、流式行为和内部参数隐藏测试。 |

## 本地开发流程

1. 配置模型服务：

```bash
python -m src.cli login
```

2. 启动网页端：

```bash
python -m src.cli web --host 127.0.0.1 --port 8081
```

3. 访问本地页面：

```text
http://127.0.0.1:8081
```

4. 修改网页端后运行：

```bash
python -m py_compile src/web/app.py
python -m pytest tests/test_web.py
git diff --check
```

## 常见改动点

### 修改前端文案

主要在 `src/web/app.py` 的 `INDEX_HTML` 中。当前页面要求：

- 中文界面。
- 面向飞行器设计。
- 模型显示为 `deepseek-v4-pro`。
- 前端只显示“飞行器设计”，不显示内部资料检索实现名。

### 修改技能行为

优先改：

```text
.clawd/skills/aircraft-conceptual-design/SKILL.md
.clawd/skills/aircraft-conceptual-design/references/workflow.md
```

修改后建议运行：

```bash
python -m pytest tests/test_aircraft_conceptual_design_skill.py
```

### 修改模型工具调用解析

优先看：

```text
src/providers/openai_compatible.py
src/providers/openai_provider.py
src/tool_system/agent_loop.py
```

修改后建议运行：

```bash
python -m pytest tests/test_providers.py
python -m pytest tests/test_agent_loop.py
```

### 修改生成数据或示范数据

生成物集中在：

```text
.clawd/generated/
```

建议同时更新 `docs/过程材料/测试与验证记录.md`，说明数据来源、脚本命令和输出文件。

## 提交过程材料规范

每次较大的 GitHub 改动建议补一份：

```text
docs/过程材料/变更记录/YYYY-MM-DD-主题.md
```

建议包含：

- 对应提交。
- 改动背景。
- 改动范围。
- 研发思路。
- 技术实现。
- 验证方式。
- 后续影响。

## 当前需要注意的边界

- 不要把内部 `aircraft-design-rag` 作为用户可见能力名。
- 不要把 RAG 参数重新暴露到前端。
- 不要展示模型隐藏思维链；只能展示工具、资料检索、耗时等可审计过程。
- 修改网页端时注意 LaTeX 渲染不要破坏代码块。
- 真实工程设计输出只能作为概念设计辅助，不能替代详细工程验证。
