# 2026-05-20 飞行器设计网页端优化

对应提交：

```text
6e0b9b3 Optimize aircraft design web UI
```

## 改动背景

项目原有网页端更偏通用聊天控制台，不足以表达“飞行器设计工作台”的专业场景。用户希望网页端使用中文、围绕飞行器总体设计任务组织信息，并支持本地资料增强、方案约束分析和工程输出。

## 改动范围

主要涉及：

```text
src/web/app.py
src/web/rag_service.py
```

## 研发思路

网页端被重新定位为“设计任务入口”，而不是模型参数面板。界面强调：

- 设计会话。
- 设计能力。
- 本地资料证据。
- 常用飞行器设计任务。
- 问答与工具活动。

前端使用中文文案，提示词围绕固定翼无人机、动力方案、低速长航时飞行器约束边界等常见总体设计任务。

## 技术实现

- 在 `INDEX_HTML` 中重写主要页面文案和布局。
- 增加飞行器设计常用问题按钮。
- 调整会话、模型、能力选择区域的视觉层次。
- 将本地资料证据以卡片方式展示。
- 对检索状态、命中数量、文件数量、片段数量、缓存状态进行摘要。

## 验证方式

建议验证：

```bash
python -m py_compile src/web/app.py
python -m pytest tests/test_web.py
python -m src.cli web --host 127.0.0.1 --port 8081
```

人工检查：

- 页面是否能正常启动。
- 中文界面是否完整。
- 常用问题按钮是否能填充输入框。
- 资料证据是否能在回答下方展开。

## 后续影响

该提交奠定了网页端领域化基础，后续“隐藏 RAG 内部实现”“默认不启用技能”“公式渲染”“过程记录可见化”等改动都是在此基础上继续细化用户体验。
