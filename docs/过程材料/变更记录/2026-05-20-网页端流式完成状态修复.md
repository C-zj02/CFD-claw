# 2026-05-20 网页端流式完成状态修复

对应提交：

```text
4b98cdb Fix web stream completion state
```

## 问题背景

网页端使用 SSE 接收流式响应。早期逻辑在收到 `done` 事件后仍可能等待连接自然关闭，导致前端停留在忙碌状态，表现为回答已经结束但按钮仍不可用。

此外，如果用户点击停止或浏览器断开连接，后端继续写入 SSE 时可能出现 BrokenPipe 或 ConnectionReset。

## 修复思路

前端：

- 收到 `event: done` 后立即记录最终 payload。
- 主动取消 reader。
- 退出读取循环。

后端：

- 对 late write 的 BrokenPipe/ConnectionReset 静默处理。
- 不把浏览器主动断开视为服务端错误。

## 影响范围

主要涉及：

```text
src/web/app.py
```

## 验证方式

```bash
python -m pytest tests/test_web.py
python -m src.cli web --host 127.0.0.1 --port 8081
```

人工验证：

- 发送问题后能看到流式输出。
- 回答完成后“发送”按钮恢复。
- 状态栏显示本轮完成。
- 点击停止后前端不崩溃。

## 研发意义

这个修复保证了网页端交互闭环：用户提问、模型流式返回、前端完成状态恢复。它也是后续“过程记录可见化”的基础，因为过程行需要准确区分处理中和已处理。
