# 2026-05-20 网页端默认不启用技能

对应提交：

```text
48ba84c Disable default web auto skill
```

## 改动背景

网页端早期可能从历史状态或默认配置中自动启用资料能力。对于普通提问，这会带来两个问题：

- 用户还没明确选择资料增强，就触发本地检索。
- 历史 `localStorage` 中的旧技能名可能影响新会话。

因此需要让网页端默认从“无技能”开始，用户需要资料增强时再主动选择“飞行器设计”。

## 改动思路

默认值调整为：

```text
autoSkill: null
```

同时清理历史前端状态中的旧选择，保证刷新或重启后不会意外恢复内部技能。

## 用户体验变化

启动网页端时：

- 顶部显示“技能：无”。
- 能力下拉默认是“不自动使用”。
- 只有用户选择“飞行器设计”后，才启用本地资料增强。

## 验证方式

建议验证：

```bash
python -m pytest tests/test_web.py
python -m src.cli web --host 127.0.0.1 --port 8081
```

浏览器检查：

- 新会话默认不启用技能。
- 刷新页面后不会自动恢复旧的内部技能名。
- 手动选择“飞行器设计”后状态能更新。

## 后续影响

该改动让前端行为更符合用户预期，也为后续“隐藏 RAG 内部实现”提供基础。
