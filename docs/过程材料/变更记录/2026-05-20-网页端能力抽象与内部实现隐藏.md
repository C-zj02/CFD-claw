# 2026-05-20 网页端能力抽象与内部实现隐藏

对应提交：

```text
2fa162a Hide RAG internals from aircraft design web UI
```

## 改动背景

底层技能名是 `aircraft-design-rag`，但这个名字暴露了内部实现。对用户来说，真正需要理解的是“飞行器设计”能力，而不是 RAG 技术细节或检索参数。

## 改动目标

- 前端只显示“飞行器设计”。
- API 对外暴露产品化能力名 `aircraft-design`。
- 后端内部仍映射到 `aircraft-design-rag`。
- RAG 参数不在前端配置。
- 会话序列化时不把内部技能名直接暴露给浏览器。

## 实现思路

后端增加产品名与内部名的映射：

```text
WEB_AIRCRAFT_SKILL_NAME = "aircraft-design"
INTERNAL_AIRCRAFT_RAG_SKILL_NAME = "aircraft-design-rag"
AIRCRAFT_SKILL_DISPLAY_NAME = "飞行器设计"
```

浏览器端只处理 `aircraft-design`，后端在创建会话和发送消息时转换为内部技能名。

## 用户体验变化

用户看到的是：

```text
不自动使用
飞行器设计
```

不会看到：

```text
aircraft-design-rag
RAG 参数
top_k
candidate_limit
max_snippet_chars
```

## 验证方式

测试覆盖：

```bash
python -m pytest tests/test_web.py
```

人工检查：

- 页面可见文本不出现 `aircraft-design-rag`。
- 选择“飞行器设计”后，后端仍能执行本地资料检索。
- API 返回的 `auto_skill` 为 `aircraft-design`。

## 研发意义

这次改动把“工程实现”与“产品能力”分离。后续即使内部检索方式变化，前端能力入口也可以保持稳定。
