# 飞行器设计功能过程材料

## 当前工作区新增过程材料

当前工作区还包含一项未提交的网页端体验改造：将回答下方的工具活动升级为类似 Codex 的“处理中/已处理 + 耗时 + 可展开过程记录”。该改动的过程材料记录在 [前端过程可见化设计说明.md](前端过程可见化设计说明.md) 中，后续提交时可作为提交说明或 PR 描述的依据。

## 使用建议

- 写 GitHub commit message 时，可以从每个 `变更记录/*.md` 的“提交说明建议”中摘取。
- 写课程/项目汇报时，可以组合 `GitHub改动过程索引.md`、`飞行器设计功能研发思路.md` 和 `测试与验证记录.md`。
- 继续迭代时，建议每次重要提交都补一份 `docs/过程材料/变更记录/YYYY-MM-DD-主题.md`，保证研发过程可追溯。
